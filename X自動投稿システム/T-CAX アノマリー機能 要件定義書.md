# T-CAX アノマリー機能 要件定義書 v1.0

**作成日**: 2026-04-01
**対象バージョン**: v8.15（予定）
**目的**: カレンダー要因（アノマリー）をシステムが自動認識し、投稿の仮説精度と品質を向上させる

---

## 背景と課題

### 発生した問題
2026年3月末、ドル円が大きく下落。AIは「円が強い（地政学リスク）」と分析したが、実際には**年度末のリパトリエーション（海外資産の円転）**というカレンダー要因が主因だった可能性が高い。

### 根本原因
AIがアノマリー（カレンダー要因による定期的な値動き傾向）を知らないため、全ての値動きをファンダメンタルズやニュースで説明しようとしてしまう。

### 解決策
「今日はどんなアノマリーが該当するか」をコードで自動判定し、プロンプト・仮説・ファクトチェックの3箇所に注入する。

## アノマリー一覧（8カテゴリ・20種類）

### カテゴリ1: ゴトー日（仲値ドル買い）

| ID | 名前 | 判定ロジック | 影響 |
|----|------|-------------|------|
| GOTO_5 | ゴトー日（5日） | 5日が営業日なら5日、休日なら直前営業日 | 東京仲値（9:55）に向けてドル買い需要。仲値後はドル売り反転の傾向 |
| GOTO_10 | ゴトー日（10日） | 同上 | 同上 |
| GOTO_15 | ゴトー日（15日） | 同上 | 同上 |
| GOTO_20 | ゴトー日（20日） | 同上 | 同上 |
| GOTO_25 | ゴトー日（25日） | 同上 | 同上 |
| GOTO_EOM | ゴトー日（月末） | 月末営業日 | 同上。ただし月末は通常のゴトー日よりフローが大きい |

**判定方法**: コードで100%自動判定。日本の祝日判定が必要（祝日シートまたは固定配列）。

### カテゴリ2: 月末・四半期末・年度末

| ID | 名前 | 判定ロジック | 影響 |
|----|------|-------------|------|
| MONTH_END | 月末リバランス | 月の最終営業日とその前日 | ロンドンフィキシング（日本時間25:00/夏時間24:00）前後に大口のリバランスフロー。方向は月初からのトレンドの逆になりやすい |
| QUARTER_END | 四半期末 | 3/6/9/12月の最終営業日とその前5営業日 | 年金・ファンドのポートフォリオリバランス。株高の四半期→株売り+債券買い+為替ヘッジ調整 |
| FISCAL_YEAR_END | 日本の年度末 | 3月25日〜31日の営業日 | 日本企業のリパトリエーション（海外利益の円転=ドル売り/円買い）。3月最終週が最も強い |
| FISCAL_YEAR_START | 日本の新年度 | 4月第1〜2週 | 新年度の海外投資再開（ドル買い/円売り）。年度末の逆フロー |

**判定方法**: コードで自動判定。カレンダー計算のみ。

### カテゴリ3: 週末金曜

| ID | 名前 | 判定ロジック | 影響 |
|----|------|-------------|------|
| FRIDAY_REBALANCE | 金曜リバランス | 毎週金曜日 | ポジション調整のリスクオフ傾向。ロンドンフィキシング前後に動きやすい。特に月末金曜は効果が増大 |

**判定方法**: 曜日判定のみ。

### カテゴリ4: 海外祝祭日（薄商い）

| ID | 名前 | 判定ロジック | 影響 |
|----|------|-------------|------|
| EASTER | イースター休暇 | 聖金曜〜翌月曜（年により変動）| 欧米の主要市場が休場。薄商いでボラ拡大リスク。ロンドン・NY不在 |
| THANKSGIVING | 感謝祭 | 11月第4木曜+翌金曜 | 米市場休場〜半日。薄商い。年末ラリーの起点になることも |
| CHRISTMAS | クリスマス休暇 | 12月24日〜26日+前後数日 | 欧米市場休場。薄商いだが「クリスマスラリー（株高=リスクオン）」の傾向あり |
| NEW_YEAR | 年末年始 | 12月29日〜1月3日 | 日米欧すべて薄商い。フラッシュクラッシュのリスク（2019年1月3日の事例） |

**判定方法**: イースターはアルゴリズムで算出（Anonymous Gregorian algorithm）。他は固定日付。

### カテゴリ5: 夏枯れ

| ID | 名前 | 判定ロジック | 影響 |
|----|------|-------------|------|
| SUMMER_LULL | 夏枯れ | 8月全体（特にお盆週） | 流動性低下でスプレッド拡大。トレンドが出にくいがフラッシュムーブのリスク |

**判定方法**: 月判定のみ。

### カテゴリ6: SQ日

| ID | 名前 | 判定ロジック | 影響 |
|----|------|-------------|------|
| MINI_SQ | ミニSQ | 毎月第2金曜 | 先物・オプション清算。株価変動→為替にも波及。特にNY時間 |
| MAJOR_SQ | メジャーSQ | 3/6/9/12月の第2金曜 | ミニSQより影響大。トリプルウィッチング（米）とも重なる |

**判定方法**: 「第N金曜日」のカレンダー計算。

### カテゴリ7: 中銀会合前後

| ID | 名前 | 判定ロジック | 影響 |
|----|------|-------------|------|
| PRE_FOMC | FOMC前日 | 経済カレンダーから自動検出 | 様子見ムード。ボラ低下→発表後に急変 |
| PRE_BOJ | 日銀会合前日 | 経済カレンダーから自動検出 | 円のボラ警戒。ポジション調整 |
| PRE_ECB | ECB会合前日 | 経済カレンダーから自動検出 | ユーロのボラ警戒 |

**判定方法**: 既存のcalendarManager.gsの経済カレンダーシートから「FOMC」「日銀」「ECB」を検出し、前日を判定。

### カテゴリ8: 月別傾向（季節性）

| ID | 名前 | 判定ロジック | 影響 |
|----|------|-------------|------|
| JANUARY_EFFECT | 1月効果 | 1月 | 新年の資金流入。リスクオン傾向。ドル円は年初に方向感が出やすい |
| SELL_IN_MAY | セルインメイ | 5月 | 「5月に売れ」。株式市場のリスクオフ傾向→安全資産の円買い |

**判定方法**: 月判定のみ。

## スプレッドシート構造

### シート名: 「アノマリー」

アノマリーの**マスターデータ**を格納するシート。コードはこのシートから影響の説明文を読み取り、日付判定はコードで自動実行する。

| 列 | ヘッダー | 型 | 例 |
|----|---------|-----|-----|
| A | アノマリーID | 文字列 | GOTO_5 |
| B | カテゴリ | 文字列 | ゴトー日 |
| C | 名前 | 文字列 | ゴトー日（5日） |
| D | 為替への影響 | 文字列 | 東京仲値（9:55）に向けてドル買い需要... |
| E | 主な通貨ペア | 文字列 | USD/JPY |
| F | 時間帯 | 文字列 | 東京（仲値前後） |
| G | 信頼度 | 文字列 | 高/中/低 |
| H | 投稿での使い方 | 文字列 | 仮説の材料として使え。「ゴトー日だから...」 |
| I | 有効 | TRUE/FALSE | TRUE |

**行数**: 20行（アノマリー種類数）。手動で1回入力すれば、以後はコードが自動参照する。

### シート名: 「祝日」（新規・オプション）

ゴトー日の営業日判定に必要。日本の祝日を年単位で登録。

| 列 | ヘッダー | 例 |
|----|---------|-----|
| A | 日付 | 2026/01/01 |
| B | 祝日名 | 元日 |

**代替案**: 祝日シートの代わりに、コード内に固定配列として定義する方法もあり。年1回の更新が必要だが、シート管理よりシンプル。

## 連携設計

### 1. プロンプト注入（promptBuilder.gs）

buildPrompt_の経済カレンダー注入の直後に配置:

```
prompt += '【📅 本日のアノマリー（カレンダー要因）】\n';
prompt += '※値動きの背景にカレンダー要因がある場合、ファンダだけで説明するな。\n';
prompt += '※アノマリーは「傾向」であり「確実」ではない。仮説の材料として使え。\n\n';
// 例: 「本日はゴトー日（25日）。東京仲値（9:55）に向けてドル買い需要が
//      入りやすい傾向。仲値通過後はドル売り反転に注意。」
```

**投稿タイプ別の注入ルール:**

| タイプ | 注入する？ | 理由 |
|--------|-----------|------|
| MORNING〜NY | ✅ 注入 | 市場系投稿はアノマリーが仮説の材料になる |
| INDICATOR | ✅ 注入 | 指標の反応がアノマリーで歪む場合がある |
| WEEKLY_REVIEW | ✅ 注入 | 週の振り返りでアノマリー要因に言及 |
| WEEKLY_HYPOTHESIS | ✅ 注入 | 来週のアノマリーを仮説に組み込む |
| NEXT_WEEK | ✅ 注入 | 来週のアノマリー予告 |
| WEEKLY_LEARNING | ❌ 不要 | 心得投稿には不要 |
| RULE系 | ❌ 不要 | 心得投稿には不要 |
| KNOWLEDGE | ⚠️ 条件付き | アノマリー自体を解説するKNOWLEDGE投稿の場合のみ |

**NEXT_WEEK / WEEKLY_HYPOTHESIS用**:
来週該当するアノマリーを先読みして注入する。getTodayAnomalies_にオフセット引数を追加し、「来週月〜金」のアノマリーを一括取得。

### 2. ファクトチェック連携（factCheck.gs）

factCheckPost_のLayer 1（システム確定データ）に追加:

```
prompt += '【確定データN: 本日のアノマリー】\n';
prompt += '本日は以下のカレンダー要因が該当します:\n';
prompt += anomalyText + '\n';
prompt += '※投稿がアノマリー要因を無視してファンダだけで説明している場合、\n';
prompt += '  ⚠️「カレンダー要因（XX）も考慮すべき」と指摘せよ。\n';
prompt += '※アノマリーを事実として断定している場合も⚠️。あくまで「傾向」である。\n\n';
```

### 3. 学び・仮説連携（learningManager.gs）

extractPostInsights_で仮説を抽出する際、アノマリー情報を文脈として渡す:

```
// 仮説の精度評価に「アノマリー要因を考慮していたか」を追加
// → 学びログに「月末リバランスを考慮した仮説」「ゴトー日を見落とした」等が蓄積
```

## 実装フェーズ

### Phase 1: 基盤（anomalyManager.gs + シート）
- アノマリーシートにマスターデータ20行を手動入力
- anomalyManager.gsの新規作成（getTodayAnomalies_ + 判定関数群）
- 祝日データの準備（config.gs内の固定配列 or 祝日シート）
- テスト関数: testTodayAnomalies_()

### Phase 2: プロンプト注入（promptBuilder.gs）
- buildPrompt_にアノマリーセクションを追加
- NEXT_WEEK/WEEKLY_HYPOTHESIS用の来週アノマリー先読み
- 投稿タイプ別の注入/非注入ルール

### Phase 3: ファクトチェック連携（factCheck.gs）
- factCheckPost_のLayer 1にアノマリー情報を追加
- 「カレンダー要因を無視している」指摘ルール

### Phase 4: 学び連携（learningManager.gs）
- 仮説抽出にアノマリー文脈を追加
- 学びログに「アノマリー考慮の有無」を記録

## 設計の鉄則（アノマリー機能固有）

1. **アノマリーは「傾向」であり「予測」ではない** — 投稿で断定させない。「ゴトー日だから上がる」はNG。「ゴトー日の仲値需要が意識される展開」がOK
2. **ファンダ > アノマリー** — ファンダメンタルズが強い日はアノマリーが効かない。両方を材料として渡し、AIに判断させる
3. **過剰注入しない** — 該当アノマリーが0件なら注入しない。3件以上なら重要度順に2件まで
4. **来週のアノマリーは予告として使う** — NEXT_WEEK/WEEKLY_HYPOTHESISで「来週はゴトー日が2回あるので仲値前後に注意」
5. **祝日データは年1回更新** — 年末に翌年の祝日を更新するリマインダーをscheduler.gsに追加

## 想定される投稿の改善例

### Before（アノマリーなし）
> 📝ドル円が159円台まで下落しました。
> →中東情勢の緊迫化を受けて、安全資産とされる円が買われている感じですね。

### After（アノマリーあり）
> 📝ドル円が159円台まで下落しました。
> →3月末は日本企業の年度末リパトリ（海外利益の円転）が出やすい時期なんですよ。
> 中東情勢のリスクオフに加えて、このカレンダー要因も円買いを後押ししている感じですね。
> →ぶっちゃけ、ファンダだけで説明しきれない動きがある時は、カレンダーを疑うのが鉄則かなと。

## 今後の拡張可能性

- **アノマリー的中率の自動集計**: 「ゴトー日のドル円上昇率は何%か」を過去のレートキャッシュから自動計算
- **アノマリー×ファンダの複合仮説**: 「ゴトー日 + 米雇用統計 = 仲値ドル買い→指標でさらに加速」
- **季節性のAI学習**: Phase 5（完全自動学習型システム）でアノマリーの有効性を自動学習
