## v12.9 コード側で全タイプに適用されるルール（C列には書かない）

以下はpromptBuilder.gs / qualityReview.gs のコード内で自動注入される。C列には記載不要。

### v12.4 から継続の基本ルール

```
【体言止めルール（★v12.4）】
  絵文字行（☕📕📝💡等）は速報ヘッドライン調の体言止め・動詞止めで書く
  OK: 「📝米イラン交渉が決裂。」「🛢原油、100ドル突破。」
  NG: 「📝米イラン交渉が決裂しました。」「〜しています。」
  →行で人間味・感想・背景を補足する

【キャラクター口調リマインダー（★v12.4）】
  プロンプト最末尾に配置。LLMは最後の指示を最も重視するため
  NG: 「市場では〇〇が後退。」← ニュース記事
  OK: 「利上げの話、正直ちょっと遠のいた感じですね。」← コンパナ

【INDICATOR対象指標（★v12.4）】
  scheduler.gsがトリガー設定時に対象指標をScriptPropertiesに保存
  buildIndicatorPolicy_が読み取り「★この指標について書け」をプロンプトに注入
  C列に記載不要（コードが自動処理）

【問いかけ（v10.0〜）】
  promptBuilder.gsが「★問いかけを入れてよい」と指示した時だけ注入
  1日2回上限、確率50%でシステムが自動判定。C列に記載不要
```

### ★v12.9 で追加されたルール

```
【GOLDEN夜の材料ヒント動的注入（★v12.9）】
  promptBuilder.gs の buildEveningMaterialHint_(postType) が動的生成
  GOLDEN投稿時のみ発動。経済カレンダーから以下を自動抽出して
  プロンプトに【夜の材料ヒント】ブロックとして注入:
    ・最重要指標（重要度「高」・雇用統計/CPI等）→ 「GOLDENでは軽く触れる程度に」
    ・中堅指標（重要度「中」・小売売上高等）→ 「GOLDENで目線を述べてよい」
    ・要人発言 → 別カテゴリで注入
  C列の「今夜の材料」セクションはこのヒントブロックがある時だけ書かれる
  材料なしの日（静かな日）はヒント空文字 → C列の「今夜の材料」セクション自動省略
  INDICATOR投稿との重複回避設計（最重要指標は「控え」程度に留める）

【Q6事実検証の確定データ優先（★v12.9）】
  qualityReview.gs のQ6プロンプトに以下を自動注入:
    ・政策金利（RBA/FRB/ECB/BOE/BOJ）
    ・通貨強弱（日次変動率）
    ・継続中重大事象（米国関税措置/米イラン対立/ホルムズ海峡関連等）
  判定優先順位を明示:
    1) 経済カレンダー確定データ
    2) 追加の確定データ（政策金利等）
    3) 本文内の数値データ
    4) Web検索（SEO影響で古い記事が上位に出やすい）
    5) Claudeの内部知識（カットオフ以前の情報）
  C列の投稿内容が確定データと矛盾すれば自動検出される
  背景: 2026-04-17 RBA誤判定事件の根本対策
```
