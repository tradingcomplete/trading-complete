# note記事 ライティングスタイルガイド

## 基本思想

### 情報の絞り方

```
相場の世界 = 材料があふれている
        ↓
全部追うと読者も自分も疲れる
        ↓
面白いと思ったテーマに絞る
        ↓
その周りが自然と見えてくる
```

**情報量が多ければいいわけじゃない。選択肢が多いと人は疲れる。**

AirPodsのレビュー記事のように、面白い材料を絞って、深く楽しく伝える。

## note最適化ルール（リサーチ結果より）

### 段落・改行ルール

| 要素 | 推奨値 | 理由 |
|------|--------|------|
| **1段落** | 3〜4行以内 | 5行超えると視覚的圧迫感 |
| **改行** | 2〜3文ごと | 意味のかたまり単位で |
| **1文** | 1〜2行 | 長くても3行まで |
| **見出し頻度** | 200〜400字ごとに1つ | 通常回1500字なら見出し4〜5個 |

### タイトルルール

| 項目 | 推奨 |
|------|------|
| **文字数** | 15〜25文字 |
| **キーワード** | 前半に配置 |
| **必須要素** | 数字 + ベネフィット |

**タイトル例：**
- ✅「ドル円3円動いた週｜僕の反省点」（19文字）
- ✅「FOMCで相場急変｜来週の戦略」（15文字）
- ❌「今週の相場についての振り返りと来週の展望について」（長すぎ）

### 投稿時間

| 時間帯 | 効果 |
|-------|------|
| **17:00〜22:00** | 最も読まれる（ゴールデンタイム） |
| **金曜18時** | 週間ピーク |
| **12:00〜13:00** | 昼休み需要 |
| **7:00〜9:00** | 通勤時間需要 |

### ハッシュタグ

**3〜5個が最適**（note公式推奨）

| 種類 | 個数 | 例 |
|------|------|-----|
| 人気タグ | 1〜2個 | #投資 #FX #個人開発 |
| ニッチタグ | 2〜3個 | #ファンダメンタルズ #トレード記録 #検証 |

### 見出し画像（サムネイル）

| 項目 | 推奨 |
|------|------|
| **サイズ** | 1280px × 670px（16:9） |
| **効果** | 画像ありはPV・スキ数が数十%UP |
| **内容** | 記事内容が一目でわかるデザイン |

### 装飾ルール

```
装飾は「引き算」が正解
```

| 装飾 | 使い方 |
|------|--------|
| **太字** | 1セクションにつき1〜2箇所のみ |
| **引用ブロック** | 他サイトからの引用、強調ポイント |
| **区切り線** | 大きなセクション間のみ |
| **箇条書き** | 手順説明、要点まとめに限定 |

**重要：余白と改行でリズムを作る > 装飾で目立たせる**

## 金融×個人開発で刺さるポイント

### ブルーオーシャンの理由

```
「金融ファンダメンタルズ × 個人開発」は
noteでまだ競合が少ない
```

- 投資系投稿は前年比164%増（追い風）
- 2025年4月「noteマネー」も立ち上げ
- 専門性 × 体験談 の組み合わせが強い

### 読者が求めているコンテンツ

1. **具体的な数字を含む実績報告**
   - 「〇〇pips獲得」「勝率〇〇%」
   - 収支データ、トレード結果

2. **失敗談・反省点の共有**
   - 「同じ失敗を避けたい」ニーズ
   - 成功談より共感を得やすい

3. **時系列での記録**
   - トレード記録の定期報告
   - 検証結果のビフォーアフター

4. **初心者向けの基本知識**
   - 専門用語の噛み砕き
   - 「要するに〜」で親しみやすく

## 記事構成パターン

1. **冒頭で「注目ポイント」を一気に提示**
2. **比較軸を明確に**（先週 vs 今週、ドル vs 円 など）
3. **各項目ごとに深堀り**
4. **他の視点・比較を随所に挿入**
5. **最後に「こんな人に読んでほしい」で読者を絞り込む**

---

## フックの作り方

| 手法 | 例 |
|-----|-----|
| **数字で具体化** | 「ドル円、今週だけで3円動いた」「5年ぶりの高値」 |
| **比較軸を明確に** | 「ドル vs ユーロ vs 円」「先週 vs 今週」 |
| **疑問形で引き込み** | 「利上げ停止？それとも継続？」「金はどこまで上がる？」 |
| **断定で興味喚起** | 「今週のMVPは金」「ドル円、ここが転換点」 |

## 口調の特徴

### 語尾パターン
- 「〜ですね」
- 「〜かと思います」
- 「〜なんですよ」
- 「〜ですしね」
- 「〜かなと」

### 親しみワード（適度に使用）
- 「マジで」
- 「めっちゃ」
- 「神」（本当に良い時だけ）
- 「やばい」

### 正直な本音を出す
- 「良くも悪くも」
- 「正直、これは微妙」
- 「〜だけで選ぶなら他にも選択肢がある」

## 人間味の出し方

### 自分の体験を具体的に
```
「今週、実際にこのタイミングでエントリーしてみたんですが...」
「僕もこのパターンで何度も負けてきました」
```

### 失敗談も隠さない
```
「正直、今週は予想が外れました」
「ここは完全に読み違えましたね」
```

### 日常シーンを描写
```
「朝のニュースチェックでこれ見た時、マジかと思いました」
「チャート見ながらコーヒー飲んでたら急に動き出して」
```

## 比較の見せ方

### 評価表で視覚化
```
| 項目 | 先週 | 今週 | 評価 |
|-----|-----|-----|-----|
| ドル円 | 150.2 | 152.8 | 📈 |
| ゴールド | 2,650 | 2,720 | 📈📈 |
```

### 不等式で序列化
```
「インパクトでいうと：雇用統計 < CPI < FOMC って感じですね」
```

### 表現パターン
- 「〜には敵わない」
- 「〜の上位互換」
- 「〜と比べると見劣りする」

---

## 専門用語の扱い

### 必ず噛み砕いて説明
```
「FOMC（米国の金融政策を決める会議）が...」
「利上げ、要するに金利を上げること、これが...」
```

### 「要するに」「つまり」で親しみやすく
```
「難しく聞こえるかもですが、要するに〜ってことです」
「つまり、ドルが買われやすい状況ってわけですね」
```

### 感覚的な表現も多用
```
「軽やかに上昇」「ガッシリした底」「ノッペリした動き」
「じわじわ上げてきた」「スコーンと落ちた」
```

---

## ネガティブ情報の伝え方

### 良い点を先に述べてからネガティブへ
```
「今週のドル円、上昇トレンドは継続中です。
 ただ、ここから上は重そうなんですよね...」
```

### 「ただ」「ただし」「けど」でつなぐ
```
「いい感じで推移してます。ただ、来週のFOMCが気になるところ」
```

### 代替案を必ず提示
```
「ドル円が難しいと感じるなら、ゴールドの方がシンプルかもです」
```

---

## セクション見出しのパターン

### 「〇〇｜□□□□」形式（結論を見出しに）
```
ドル円｜150円突破、ここからが本番
ゴールド｜2,700ドル超え、まだ上がる？
来週の注目｜FOMCで全てが決まる
```

### 疑問形で引く
```
利上げ停止？それとも継続？
金はどこまで上がるのか
```

### 断定で興味喚起
```
今週のMVP｜間違いなくゴールド
注目イベント｜FOMCに全集中
```

---

## 記事の締め方

### 「まとめると」で箇条書き
```
まとめると：
- ドル円は上昇トレンド継続
- ただし152円台は重い
- 来週のFOMCが転換点になりそう
```

### 良い点/悪い点を明確に分離
```
【良かった点】
- 雇用統計の結果でトレンド確認できた

【反省点】
- 利確が早すぎた
```

### 「という感じで」でカジュアルに
```
「という感じで、今週はこんな相場でした」
```

### 行動喚起で締める
```
「LINE登録で検証ツールもらえるので、よければ👇」
```

## 金融記事への応用まとめ

### 冒頭フック
- 「今週のドル円、〇〇が2倍のインパクト」
- 「金価格、5年ぶりの高値更新」
- 数字で具体化して引き込む

### 比較軸の設定
- 「ドル vs ユーロ vs 円」
- 「金 vs 株 vs 仮想通貨」
- 「先週 vs 今週 vs 来週予測」

### セクション構成例
```
今週の注目ポイント｜〇〇が相場を動かす
ドル円｜〇〇で上昇？下落？
ゴールド｜〇〇の影響で〇〇
来週の注目イベント｜〇〇に要警戒
まとめ｜今週の学びはこれ
```

## チェックリスト

記事を書いたら確認：

```
□ テーマを絞れているか（全部盛りになっていないか）
□ 冒頭で「今週の注目ポイント」が明確か
□ 専門用語に説明を添えているか
□ 自分の体験・本音が入っているか
□ 見出しに結論が入っているか
□ ネガティブ情報は代替案とセットか
□ 「という感じで」的なカジュアルな締めになっているか
□ LINE誘導のCTAがあるか
```

【ライター特性分析】AirPods Pro 3 レビュー記事

【記事構成パターン】
1. 冒頭で「進化ポイント」を箇条書きで一気に提示
2. スペック比較表で競合と並べて客観性を出す
3. 各項目ごとに深堀り（装着感→音質→ノイキャン→外音取り込み→操作性）
4. 他社製品との比較を随所に挿入
5. 最後に「こんな人におすすめ」で読者を絞り込む

【フックの作り方】
- 数字で具体化：「2倍のノイズキャンセリング」「初代比4倍」
- 比較軸を明確に：「AirPods Pro 2 vs 3 vs 4」の三つ巴
- 疑問形で引き込み：「空間オーディオの変化は？」「通話品質｜変わらない？」
- 断定で興味喚起：「圧倒的NO.1に」「相変わらず神」

【口調の特徴】
- 語尾パターン：「〜ですね」「〜かと思います」「〜なんですよ」「〜ですしね」
- 親しみワード：「神」「マジで」「めっちゃ」「イイトコどり」
- 正直な本音：「良くも悪くも」「敵なし」「音質だけで選ぶなら他にも選択肢がある」

【人間味の出し方】
- 自分の体験を具体的に：「東京出張にAirPods Pro 3を連れて行って」
- 失敗談も隠さない：「僕も初代AirPods Proを一度無くしている」
- 日常シーンを描写：「家の中でAirPods Proを見失うことが結構多い」
- 妻との生活をチラ見せ：「週一で妻と一緒にホットヨガに通ってます」

【比較の見せ方】
- 評価表で5段階評価（視覚的にわかりやすい）
- 不等式で序列化：「AirPods Pro 2 < WF-1000XM5 < AirPods Pro 3 < Bose」
- 「〜には敵わない」「〜の上位互換」などの表現

【読者への問いかけ方】
- 「お兄さんはどんなのがすき？」的な直接質問はなし
- 代わりに「〜な方にとっては」「〜用途には向いていない」で読者を選別
- 「よければチェックしてみてくださいね！」で行動促進

【専門性と親しみやすさのバランス】
- 専門用語は使うが必ず説明を添える
- 「要するに」「つまり」で噛み砕く
- 感覚的な表現も多用：「軽やか」→「ガッシリ」「ノッペリ感」

【ネガティブ情報の伝え方】
- 良い点を先に述べてからネガティブへ
- 「ただ」「ただし」「けど」でつなぐ
- 代替案を必ず提示：「音質を最優先したい場合はEAH-AZ100や...をおすすめ」

【セクション見出しのパターン】
- 「〇〇｜□□□□」形式：「音質｜AirPods Pro 2よりもメリハリ感のある音に」
- 結論を見出しに入れる：「外音取り込み｜圧倒的NO.1に」「操作性｜相変わらず神」
- 疑問形も使う：「ライブ翻訳機能は現状だと実質使えない」（否定形で興味喚起）

【数値・データの使い方】
- スペック表で網羅的に
- 本文中では「〜時間」「〜倍」「〜dB」など具体数字
- 「約」「くらい」で正確さを担保しつつ親しみやすく

【記事の締め方】
- 「まとめると」で箇条書き
- 良い点/悪い点を明確に分離
- 「という感じで」でカジュアルに
- 購入リンクで行動喚起
