## 1. TCが戦う市場の定義

### レッドオーシャン（TCが戦わない場所）

「トレードを簡単に記録し、自動集計するアプリ」という切り口での戦いは避ける。
国内外に多数のSaaSツール・FX業者の無料アプリ・無料のExcelテンプレートが乱立しており、価格競争に巻き込まれる。

### ブルーオーシャン（TCが戦う場所）

「事前の規律を構築する」「エントリー前にトレーダーを立ち止まらせる」というポジショニングにおいて、国内外を見渡しても競合が1つも存在しない完全なブルーオーシャン。

さらに広告面でも、「FX 管理 アプリ」「トレード記録 アプリ」といったニッチキーワードには大手FX業者が参入しておらず、Google検索広告でもブルーオーシャンが形成されている。

## 2. TCの競争優位性（なぜ勝てるか）

### 理由1: FX業者には構造的に絶対に真似できない「強制チェックリスト」

FX業者の利益はトレーダーの取引回数（スプレッド）に依存している。
そのため業者が提供する無料ツールは「いかに簡単に何度もトレードさせるか」に特化している。

エントリー前に3つの根拠を言語化させるような「トレードを思いとどまらせる摩擦」は、業者のビジネスモデルと完全に逆行する。彼らが自ら実装することは絶対にない。

TCのこの設計思想は、業者ツールに対する**永続的な競争優位性**になる。

### 理由2: DIYのAI分析では到達できない「計算式からAIへ」の二段階アーキテクチャ

多くのトレーダーが「ExcelデータをそのままChatGPTに分析させればいい」と考える。
しかし生データの集計を直接AIに任せると、計算ミスやコンテキストの限界に突き当たる。

TCはシステム側で勝率・プロフィットファクター・ルール遵守率などを正確に計算し、その整頓された統計データの傾向をAIに読み解かせる。非常に安定した改善提案が可能になる。

日本語環境でこの精度のAI分析と規律構築を掛け合わせたツールは他にない。

### 理由3: 「簡単に記録できても勝てないと痛感した層」の強力な受け皿

日本のFXトレーダーの多くが「Excelで記録はしているが、どう分析していいかわからない」という壁にぶつかっている。

簡単に入力できるアプリを使って負けのループを繰り返した経験のある30代〜40代のサラリーマン層にとって、「簡単に記録するツールを探しているならTCは合いません」という突き放したメッセージと、「記録してからトレードする」というTCの厳しさは、本気で状況を変えたい彼らの心に深く刺さる。

### TCの本質（最重要メッセージ）

```
すべての競合が「もっと簡単に」「もっと速く」と叫ぶ中で、
TCだけが「立ち止まれ」と伝える逆張りこそが最大の本質。
```

## 4. 広告媒体の選定と優先順位

### 5媒体の比較評価

| 媒体 | 日本MAU | 30代男性リーチ | CPC相場 | FX規制リスク | 優先度 |
|------|---------|--------------|---------|------------|--------|
| X（Twitter） | 6,800万 | 最高 | 20〜200円 | 低 | 最優先 |
| Google検索 | シェア75% | 高（顕在層） | 80〜500円 | 中〜高 | 第2優先 |
| YouTube | 7,000万 | 高 | CPV 3〜20円 | 中〜高 | 後回し |
| Meta（FB/IG） | FB 2,600万 | 中 | 50〜300円 | 中〜高 | 後回し |
| LINE | 1億超 | 高 | 24〜200円 | 低〜中 | 不採用 |

### X（Twitter）が最優先の理由

FXトレーダーコミュニティがXに最も密集している。#FX・#ドル円・#トレード記録などのハッシュタグが日常的に飛び交い、副業トレーダーが匿名で情報交換する主戦場だ。

CPCが平均50〜80円と金融系にしては安価で、リポスト（RT）されたツイートからのアクションは課金対象外のため、拡散されるほど実質コストが下がる。T-CAXアカウントの運用実績をそのまま活かせる点も大きなアドバンテージ。

### YouTubeが後回しの理由

動画クリエイティブの制作コスト（10〜50万円）が別途必要で、月3万円以下の予算と噛み合わない。

### LINEを不採用とした理由

リーチ力は圧倒的だが、「FXトレーダー」というニッチ層へのターゲティング精度が不十分で費用対効果が出にくい。

## 5. X広告の具体的戦略

### ターゲティング設計

#### フォロワーターゲティング（最重要）

特定アカウントのフォロワーに似た興味関心を持つユーザーに配信する、X固有の機能。25アカウント以上の設定が推奨される。

| カテゴリ | 具体的なアカウント候補 |
|---------|-------------------|
| 個人トレーダー系 | 羊飼いFX・小林芳彦（JFX社長）・バカラ村・後藤達也 など |
| FX会社系 | 楽天FX・SBIリクイディティ・マーケット など |
| 経済メディア系 | 日経電子版マーケット・ロイター日本語 など |

FXブローカーのフォロワーは「実際に口座を開いてトレードしている人」であり、記録ツールSaaSとの親和性がきわめて高い。

#### キーワードターゲティング（25個以上設定推奨）

| カテゴリ | キーワード例 |
|---------|------------|
| 課題直結型（高CVR期待） | トレード記録・トレード日記・トレードノート・振り返り・トレード管理 |
| ペインポイント系（高CVR期待） | 損切り・FX勝てない・ポジポジ病・ルール守れない・メンタル管理 |
| 通貨ペア・手法系 | ドル円・ユーロドル・スキャルピング・デイトレード・チャート分析 |

除外キーワード: 「FX 詐欺」「FX 危険」は必ず設定する

### クリエイティブの方針（最重要）

TCの逆張りメッセージを前面に出す。「簡単に記録できます」という言葉は使わない。

```
NG: 「トレードを簡単に記録できます」→ レッドオーシャンの言葉
OK: 「簡単に記録するツールを探しているならTCは合いません」
OK: 「エントリー前に3つの根拠を書けないなら、入ってはいけない」
OK: 「Excelでのトレード記録、まだ続けますか？」
```

フォーマット: プロモ広告（ウェブサイトクリック目的）+ チェックリスト画面のスクリーンショットまたは15〜30秒のデモ動画

### X広告の費用感

| 月予算 | 推定クリック数（CPC80円想定） | 評価 |
|--------|---------------------------|------|
| 3,000円 | 約37クリック | データ蓄積が困難 |
| 10,000円 | 約125クリック | A/Bテスト可能 |
| 30,000円 | 約375クリック | 本格運用ライン |

## 6. Google検索広告の具体的戦略

### 狙い目キーワード一覧

大手FX業者が参入していないニッチキーワードに絞る。

| キーワード | 推定月間検索数 | 推定CPC | 検索意図 | 優先度 |
|-----------|-------------|---------|---------|--------|
| FX 管理 アプリ | 300〜800 | 100〜250円 | ツール探し（高CV） | 最優先 |
| トレード記録 アプリ | 200〜600 | 80〜200円 | ツール探し（高CV） | 最優先 |
| FX 記録 続かない | 不明（ロングテール） | 50〜100円 | ペインポイント（高CV） | 高 |
| トレード日誌 | 500〜1,500 | 50〜150円 | 情報収集＋ツール探し | 高 |
| FX エクセル 管理 面倒 | 不明（ロングテール） | 50〜100円 | ペインポイント | 中 |
| Myfxbook 代替 | 不明（ロングテール） | 50〜100円 | 比較検討層 | 中 |

マッチタイプ: 完全一致のみ使用（無駄クリック防止）

配信時間帯: 平日19:00〜24:00・土日終日（サラリーマントレーダーの活動時間）

### Google広告の規制対策（重要）

TCはFXブローカーではなく記録ツールのため、金融商品取引法の広告規制対象外。
ただし、Googleの自動審査でFX関連と誤認され不承認になるリスクがある。

| 対策 | 具体的な内容 |
|------|------------|
| LP上の免責事項 | 「当サービスは金融商品ではありません」「投資助言は行いません」を明記 |
| 広告文の表現 | 「FXで勝てる」「利益が増える」などの表現を一切使わない |
| 訴求軸 | 「トレード記録の効率化」「振り返りツール」という機能訴求に徹する |
| 不承認時の対応 | 手動審査をリクエストする |

## 7. 予算配分プラン

### 許容CPA（顧客獲得単価）の上限設定

```
Proプラン月額1,980円 × 平均継続6ヶ月 = LTV 約11,880円
許容CPA上限: 約4,000円
```

この数字を常に意識して広告運用する。CPAがこれを超えているチャネルは改善または停止する。

### 予算帯別の配分

| 月予算 | X広告 | Google検索 | リターゲティング | 目的 |
|--------|------|-----------|--------------|------|
| 3,000円 | 3,000円 | なし | なし | クリエイティブの傾向把握 |
| 10,000円 | 7,000円 | 3,000円 | なし | A/Bテスト開始・CVRの基礎データ取得 |
| 20,000円 | 12,000円 | 8,000円 | なし | 2チャネル本格テスト |
| 30,000円 | 18,000円 | 10,000円 | 2,000円 | 最適化フェーズ・リターゲティング開始 |

## 8. 3ヶ月のテスト計画

### 1ヶ月目: 基盤構築（広告費なし）

```
- LPの免責事項を整備（「金融商品ではありません」を明記）
- XピクセルをLPに設置（リターゲティング準備）
- Google広告の審査申請（金融商品との誤認を事前に防ぐ）
- XでFXトレーダー向けオーガニック投稿を開始（広告前の認知形成）
- noteでリリース記事・開発ストーリー記事を公開
- Product Huntへの登録
```

### 2〜3ヶ月目: テスト運用（月2〜3万円）

```
X広告:
  - フォロワーターゲティング vs キーワードターゲティングを別キャンペーンで比較
  - クリエイティブ2パターンのA/Bテスト実施
  - どちらのCPAが低いかを検証

Google広告:
  - キーワード3〜5本（完全一致）で配信
  - 配信時間帯を平日夜間・週末に限定
  - CPA実績データを蓄積
```

### 4ヶ月目以降: データに基づく最適化

```
- 許容CPA（4,000円）以下のチャネルに予算を集中
- 超えているチャネルは停止または改善
- SEO記事・オーガニック発信を育て、広告依存度を段階的に下げる
```

## 9. 個人開発SaaSの広告に関する重要な事実

調査の結果、**個人開発SaaSが有料広告のみで成功した事例はきわめて少ない**ことが判明している。

| 事例 | 結果 |
|------|------|
| 個人開発アプリのXプロモーション（予算1万円・2日間） | 約70DL・1DLあたり143円 / 「限定的な効果」と開発者が総括 |
| Spotify WordCloud のTwitter広告（予算1,000円） | 16クリック・実際に使ったユーザーは2名のみ |
| 海外IndieHacker Google Ads（予算$100） | 174クリック・28サインアップ（CVR16%）/ 「広告とindie hackingは相性が良くない」 |

成功した個人開発者（Inkdrop・Pomofocus等）はいずれも、**SEOとコンテンツマーケティングを主力**に据えており、有料広告はあくまで補完手段として位置づけている。

### 広告の正しい位置づけ

```
基盤: SEO（「FX トレード記録の書き方」などの記事）
主力: X（T-CAXアカウントのオーガニック発信）
補完: 有料広告（データを取りながら段階的に拡大）
```

リリース直後は広告より発信に時間を使う方がコスパが高い。ある程度口コミが生まれてから有料広告を開始するのが、個人開発SaaSとして持続可能な戦略だ。
