## 6. 法務・コンプライアンス

### 金融商品取引法との関係

**Trading Completeは金商法の規制対象外**:
- 投資助言ではない（過去データの分析のみ）
- 金融商品の販売・媒介ではない（記録ツール）
- 投資顧問業ではない

ただし**自動審査（決済プロバイダ等）では「FX」キーワードで一括拒否される構造**があるため、対外文書では「FX」「金融」「投資」を控えめに、「トレード記録」「データ分析」「規律」を前面に出す。

### 投資助言にならない表現ルール

詳細: `docs/marketing/Trading_Complete_最終収益モデル戦略` のAI機能ガイドライン (archive行きだが内容は有効) を参照。

✅ OK: 「過去90日のデータでは月曜日の勝率が60%でした」
❌ NG: 「今エントリーすべき」「このシグナルは勝てる」

## 7. データ・APIキー管理

### API キー類のルール

**絶対NG**:
- git に本番キーをコミット
- ドキュメントに本番キーを書く
- スクリーンショットに本番キーを写す

**OK**:
- Supabase Edge Functions Secrets に登録
- `.env.local`（gitignore済み）にローカル保存
- 1Password / Bitwarden 等の専用管理ツール

**現在のSecrets**:
- Supabase Dashboard で管理（PayPal / Square 関連各種）
- 詳細は `docs/features/決済システム要件定義書_v3_10.md` §3 / Step 4

### 個人情報の取り扱い

- ユーザーデータは Supabase に格納（RLS でユーザー別分離）
- Phase 11 でサーバー側プラン制限が完成したらリリース可
- データ削除リクエストへの対応フロー整備（要件定義 v0.2 で詳細追記）

### バックアップ
- Supabase は自動バックアップ（無料プランは7日間）
- 重要データは月1回 ローカル CSV エクスポート（手動）
- ソースコードは GitHub（origin/main）

## 8. リリースサイクル

### 機能リリースの基本フロー

1. 要件定義書を `docs/features/` に作成
2. Phase 分けで実装（v3.10 のような構造化）
3. テスト完了 → `docs/TASKS.md` を ✅ 更新
4. `docs/REFERENCE.md` に完了履歴・教訓を追記
5. 必要に応じて LP / 記事 / X で告知（`/cross-update` スキル活用）

### バージョニング

- アプリ本体: 機能ごとに `v1.x.x` で内部管理（リリース時に v1.0 確定）
- ドキュメント: 「常に最新」前提・大改訂時のみ archive
- API: 後方互換性を最優先・破壊的変更は v2.0 以降に集約

### 次のメジャーマイルストーン

| マイルストーン | 時期 | 残作業 |
|---|---|---|
| 正式リリース v1.0 | 2026年夏 | Square Phase 10 + Phase 11 |
| Premium追加 v1.1 | 2026年9月 | AI改善提案機能 |
| グローバル展開 v3.0 | 2027年以降 | i18n / 基準通貨 |

## 10. 緊急時対応

### 想定リスクとプレイブック（骨子）

| 危機 | 想定影響 | 一次対応 | 詳細プレイブック |
|---|---|---|---|
| 決済プロバイダ閉鎖 | サブスク停止 | 多層決済戦略により別プロバイダで継続 | （要追記・実例: 2026-03-17 Stripe閉鎖） |
| Supabase障害 | DB停止 | エラーメッセージ表示・ステータスページ告知 | （要追記） |
| データ漏洩 | 信頼失墜・法的責任 | 即時アクセス遮断・影響範囲特定・ユーザー通知 | （要追記） |
| X凍結 | 主要マーケティングチャネル喪失 | 別アカウント立ち上げ・noteとYouTubeにリーチ依存 | （要追記） |
| ソース紛失 | 開発停止 | GitHub origin から復元 | 自動 |
| 屋号変更要求 | ブランド再構築 | /cross-update で全リポ波及対応 | スキルで対応可能 |
| 法的クレーム | 訴訟リスク | 顧問弁護士相談（成長期に契約検討） | （要追記） |

### 連絡網
- 社長: コンパナ（X DM / メール）
- 顧問弁護士: 未契約（年商増加時に検討）
- 顧問税理士: 未契約（確定申告期に検討）
- Supabase / Square / PayPal サポート: 各ダッシュボードから

### 教訓
- 「1社に全部賭けない」（多層決済戦略の根拠 / v3.10 §5 教訓25）
- リスクは事後対応より事前準備が安い
- 過去の失敗は `docs/REFERENCE.md` に記録・共有
