### 初期化タイミング（Phase 4）

### イベント重複登録

**対処**: cloneNodeパターン
```javascript
const newInput = pairInput.cloneNode(true);
pairInput.parentNode.replaceChild(newInput, pairInput);
```

### iOSフォント自動ズーム（2026-01-24追加）

**原因**: `font-size` が 16px 未満の入力フィールドをタップするとiOSが自動ズーム

**対処**: 入力フィールドの `font-size` を最低 16px に設定
```css
/* 6_responsive.css */
@media (max-width: 768px) {
    input, select, textarea {
        font-size: 16px; /* 16px未満にしない */
    }
}
```

**影響ファイル**: `6_responsive.css`（設定タブ・セルフイメージ入力欄）

### CSSアニメーションのブラウザ互換性（2026-02-25追加）

**原因**: `background-clip: text` と CSS `@keyframes` の組み合わせがSafari（iOS含む）で動作しない

**対処**: `requestAnimationFrame` を使ったJSアニメーションに変更
```javascript
// ❌ CSSアニメーション（Chrome専用）
@keyframes shimmer { ... }

// ✅ JS実装（全ブラウザ対応）
function animateShimmer(timestamp) {
    // requestAnimationFrameで毎フレーム更新
    requestAnimationFrame(animateShimmer);
}
```

**影響ファイル**: ヘッダーシマーアニメーション（script.js）

### localStorageデータ形式不整合（2025-12-23追加）

**原因**: 保存時のデータ形式が後から変わった場合、読み込み時に旧形式データが混在する

**対処**: 読み込み時に後方互換性のマイグレーション処理を入れる
```javascript
// データ読み込み時に形式を検証・変換
function migrateTradeData(trade) {
    // 旧形式 → 新形式への変換
    if (!trade.exits) trade.exits = [];
    return trade;
}
```

**影響ファイル**: TradeEdit.js、TradeEntry.js

## 7. トレードステータス判定（2025-11-29追加）

### 判定ロジック

```javascript
// TradeDetail.js での判定
const hasExits = trade.exits && trade.exits.length > 0;
const isClosed = trade.holdingStatus === 'closed';
const isOpen = !hasExits && !isClosed;
const isSettled = hasExits || isClosed;
```

### ステータス一覧

| ステータス | 条件 | 説明 |
|-----------|------|------|
| `isOpen` | 決済なし & クローズでない | 保有中のトレード |
| `isSettled` | 決済あり or クローズ | 決済済みのトレード |
| `hasExits` | exits配列に要素あり | 決済情報が存在 |
| `isClosed` | holdingStatus === 'closed' | 明示的にクローズ |

### ノート保存がリロードで消える（2025-11-27）🔴 CRITICAL

**症状**: ノートを保存してもリロードするとデータが消える

**原因**: NoteManagerModule.js の `#save()` メソッドで `window.saveNotes()` を呼んでいた

```javascript
// ❌ 問題のコード
#save() {
    localStorage.setItem('notes', JSON.stringify(this.#notes));
    window.notes = this.#notes;
    if (typeof window.saveNotes === 'function') {
        window.saveNotes();  // ← これが古いデータで上書き！
    }
}

// ✅ 正しいコード
#save() {
    localStorage.setItem('notes', JSON.stringify(this.#notes));
    window.notes = this.#notes;
    // window.saveNotes() は呼ばない
}
```

**教訓**: モジュール化時に過渡期の互換性コードを残す場合、保存処理の競合に注意

### カレンダー⇔週間プレビュー連動しない（2025-11-27）

**症状**: カレンダーで日付を選択しても週間プレビューが更新されない

**原因**: `noteDate` の `onchange` で `loadNoteForDate()` のみ呼んでいた

```html
<!-- ❌ 問題のコード -->
<input type="date" id="noteDate" onchange="loadNoteForDate(this.value); updateMonthlyMemoDisplay();">

<input type="date" id="noteDate" onchange="selectNoteDate(this.value); updateMonthlyMemoDisplay();">
```

**教訓**: 複数UIの連動は、統合的なメソッドを用意して一箇所で制御

## 11. 画像データ構造（2026-01追加）

`imageUtils.js v1.3.0` で管理する画像オブジェクトの形式：

```javascript
{
  data: "base64...",        // ローカル画像（base64）
  path: "user_id/xxx.jpg", // Supabase Storageパス
  url: "https://...",      // 署名付きURL（有効期限あり）
  urlExpiry: 1700000000,   // URL有効期限（Unixタイムスタンプ）
  title: "画像の題名",      // 画像キャプション題名（2026-01-17追加）
  description: "説明文"    // 画像キャプション説明（2026-01-17追加）
}
```

### 署名付きURL取得（期限切れ自動更新）

```javascript
// ❌ urlを直接参照すると期限切れで表示されない
img.src = imageData.url;

// ✅ 期限チェック付きで取得
const url = await window.getValidImageSrc(imageData);
img.src = url;
```

| 関数 | 説明 |
|------|------|
| `window.isUrlExpired(img)` | URLが期限切れか判定（true = 期限切れ） |
| `window.getValidImageSrc(img)` | 期限切れなら自動更新して有効なURLを返す |
