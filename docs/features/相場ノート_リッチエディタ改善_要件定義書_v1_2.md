## 📍 背景・課題

### 課題1: 文字サイズ変更が2回目で反応しない

**現象**:
1. テキストを選択して「小」を適用 → 正常
2. 同じテキストを再度選択して「小」を適用 → 反応しない
3. 一度「中」にしてから「小」を選ぶ必要があった

**原因**:
既存の`applyFontSize()`関数が、選択範囲に新しい`<span>`を追加するだけで、既存のフォントサイズを解除していなかった。

---

### 課題2: em単位でネスト時にサイズが変化する

**現象**:
- 大 → 小 を押すと、一度「中」くらいのサイズになる
- 大 → 大 を何回も押すとどんどん大きくなる

**原因**:
em単位は**相対値**のため、spanがネストすると掛け算になる。

```
大(1.6em) × 小(0.9em) = 1.44em（中くらいに見える）
大(1.6em) × 大(1.6em) = 2.56em（巨大化）
```

---

### 課題3: 文字サイズの差が分かりづらい

**現象**:
小(0.9em)、中(1.1em)、大(1.3em)の差が視覚的に小さい。

**原因**:
サイズ差が0.2em刻みで、見た目の変化が少なかった。

---

### 課題4: コピー&ペーストでHTMLタグが混入

**現象**:
Webページからテキストをコピーして貼り付けると、元のHTML（フォント、色、リンク等）がそのまま入ってしまう。

**ユーザーの回避策**:
メモ帳などに一度貼り付けてから再コピーする必要があった。

**原因**:
`contenteditable`のデフォルト動作はHTML形式での貼り付け（ブラウザの仕様）。

---

### 課題5: 色を付けた後にサイズ変更すると色が消える

**現象**:
テキストに赤色を適用した後、文字サイズを変更すると色がリセットされる。

**原因**:
v3の実装で`textContent`（プレーンテキスト）を使用していたため、全ての装飾が失われた。

---

### 課題6: 改行後もサイズが引き継がれる

**現象**:
「大」サイズで文字を入力した後、Enterで改行すると次の行も「大」のまま。デフォルトに戻す手間がかかる。

**原因**:
contenteditableの仕様で、改行時に直前のスタイルを引き継ぐ動作になっている。

**ユーザーの実態**:
基本はデフォルト（小）で入力し、目立たせたい部分だけを「大」にする使い方が多い。

---

## ✅ 解決策

### 解決策1: applyFontSize関数の改善（v4）

**方針**: 
1. 選択範囲のHTMLを抽出（`extractContents`）→ 装飾は保持
2. 抽出したHTML内の**font-sizeスタイルだけ**を除去（色などは保持）
3. 新しいspan(font-sizeのみ)で囲んで挿入
4. エディタ全体から空spanを除去

---

### 解決策2: px単位（絶対値）への変更

**方針**: em（相対値）からpx（絶対値）に変更し、ネストしても影響を受けないようにする。

| サイズ | 変更前 | 変更後 | 説明 |
|--------|--------|--------|------|
| 小 | 0.9em | **16px** | デフォルトと同じ |
| 中 | 1.1em | **20px** | +4px |
| 大 | 1.3em | **24px** | +8px |

**効果**:
- 大→小を1回押すだけで16pxになる
- 大を何回押しても24pxのまま
- 小中大以外のサイズにならない
- 「小」はデフォルトサイズと同じなので自然

---

### 解決策3: スマートペースト機能

**方針**: ペースト操作を2種類に分離

| 操作 | 動作 | 用途 |
|------|------|------|
| **Ctrl+V** | プレーンテキストで貼り付け | 外部サイトからコピー（装飾除去） |
| **Ctrl+Shift+V** | HTMLそのまま貼り付け | 同じノート間で移動（装飾維持） |

---

### 解決策4: プレースホルダーに操作説明追加

**方針**: ユーザーが機能を発見できるよう、入力欄が空の時に操作説明を表示

**変更前**:
```
今日の相場の動き、出来事、気づいたことなど（複数行入力可能）
```

**変更後**:
```
今日の相場の動き、出来事、気づいたことなど ／ Ctrl+V: テキスト貼付 ／ Ctrl+Shift+V: 装飾維持
```

---

### 解決策5: Enter改行でスタイルリセット

**方針**: Enterキーで改行した際、次の行を**デフォルトサイズ（16px）に自動リセット**する。

| 操作 | 動作 |
|------|------|
| **Enter** | 改行 + スタイルリセット（デフォルトに戻る） |
| **Shift+Enter** | 改行のみ（スタイル引き継ぎ） |

**使用例**:
```
本日のドル円は上昇。          ← デフォルト(16px)
【重要】雇用統計発表！        ← 大(24px)で入力
                              ← Enter押下
明日も注目。                  ← 自動でデフォルト(16px)に戻る
```

## 🔧 技術仕様

### NoteManagerModule 変更メソッド

| メソッド | 変更内容 |
|---------|---------|
| `applyFontSize()` | v4改善版（装飾保持、px単位） |
| `setupNoteAutoSave()` | pasteイベント追加、keydownイベント追加 |

### NoteManagerModule 追加メソッド

| メソッド | アクセス | 説明 |
|---------|----------|------|
| `#removeFontSizeOnly(node)` | Private | ノード内のfont-sizeスタイルだけを再帰的に除去 |
| `#removeEmptySpans(editor)` | Private | エディタ内の空spanを除去 |

### sizeMap（フォントサイズ定義）

```javascript
const sizeMap = {
    'small': '16px',   // デフォルトと同じ
    'medium': '20px',  // +4px
    'large': '24px'    // +8px
};
```

### setupNoteAutoSave イベントリスナー

| イベント | 処理 |
|---------|------|
| `input` | 自動保存タイマー設定 |
| `blur` | 即時自動保存 |
| `keydown` | Enterでスタイルリセット改行 |
| `paste` | スマートペースト（Ctrl+V / Ctrl+Shift+V） |

### EventBus

既存イベントを維持（変更なし）:
- `note:fontSizeApplied` - フォントサイズ適用時

## 💡 学んだこと

| 項目 | 内容 |
|------|------|
| em単位の罠 | 相対値なのでネストすると掛け算になる。絶対値（px）を使うべき |
| extractContents | 選択範囲のHTMLを抽出するが、外側のタグは残る |
| textContent | プレーンテキストのみ取得。装飾は全て失われる |
| contenteditable | デフォルトでHTML貼り付け。pasteイベントで制御が必要 |
| デフォルトサイズ | 「小」をデフォルト(16px)と同じにすることで自然な操作感 |
| 改行時のスタイル | contenteditableは改行時にスタイルを引き継ぐ。keydownで制御が必要 |
