## 1. エグゼクティブサマリー

### 1.1 v1.3 の核心メッセージ

> **TC Premium は、副業FXトレーダーの「業務OS」である。**
> 記録、分析、確定申告、経費管理、翌年の戦略——
> 1年サイクルで回り続けるトレーダー業務を、AIが裏側で支える。
> ユーザーが切ろうとしたとき、過去のデータを失うコストが乗り換えコストを超える。
> それが「敵に回すと困るサービス」の正体である。

### 1.2 v1.2 から v1.3 への思想的進化

**v1.2の位置づけ**:
- AIトレードコーチが Premium の中心
- 「記録が資産になる瞬間」を作る
- 4 Phase 構成で機能を順次拡張

**v1.3の位置づけ**:
- **確定申告AI** が Premium の中心(日本特化の参入障壁)
- 「**コストではなく自己投資**」と認識される SaaS を作る
- **不景気で切られないサービス** を構造的に設計
- 機能数の足し算ではなく、**シンプルさと不可欠さ** で勝負
- リリース後の行動データで継続改善するSaaSの王道に乗る

### 1.3 一文まとめ

> 「あなたのトレードと確定申告を、AIが1年サイクルで支え続ける。
> その記録は、あなた自身の資産になる。」
> それが Premium の約束。

## 2. リサーチ要旨

2026年4月、A(競合AI実態) + B(トレーダー生の声) + E(行動変容理論)の3軸で実施したリサーチの結論。詳細は別レポート参照。

### 2.1 「ないと困る」AI機能の3条件(必須充足)

1. **受動性(Passive Delivery)**: ユーザーが質問しなくても勝手に届く
2. **行動可能性(Actionability)**: 次のトレードで何を変えるかが具体的
3. **金額換算(Yen Translation)**: 抽象的な改善点を金額で語る

この3条件を全て満たすのは、Edgewonk Edge Finder と TraderSync Cypher Coach のみ。「Cypher Coach相当のプロアクティブ通知」を含めることが TC Premium の価格(¥2,980/月)の正当化の核心。

### 2.2 日本人副業FXトレーダーの悩みTOP10(頻度順)

1. 記録は続けたいけど続かない/面倒
2. 記録はしているけど活かせていない
3. 分析の仕方が分からない
4. 自分の負けパターンが分かれば...
5. 副業ゆえの時間不足
6. メンタル/感情/ティルト
7. 次のトレードで気をつけるべきこと
8. プロと自分の何が違うのか
9. ジャーナリングを続けても勝てない
10. **確定申告の面倒さ**

→ 1〜9はAIトレードコーチで対応、**10は確定申告AIで対応**(日本特化、海外SaaSが触れない領域)

### 2.3 行動変容のテキスト設計

- **Losada / Gottman 比** 肯定:否定 = 3:1〜5:1 を厳守
- **Fogg Tiny Habits** Anchor + Tiny Behavior + Celebration の3点セット
- **CBT原則** 「行動を責めて人格を責めない」「事実→解釈→次回のリフレーム」
- **Mark Douglas整合** 予測ではなくプロセス、確率の語り

詳細は §10 を参照。

### 2.4 不景気で切られないSaaSの条件

| 条件 | TC Premium への適用 |
|------|------------------|
| 失うコスト > 乗り換えコスト | 過去3年分の損益繰越・経費・プレイブックを失う構造 |
| 業務フローに溶け込む | 1年サイクル(記録→分析→申告→翌年)に組み込む |
| お金を生む/守るサービス | 節税額・回避損失・時間節約を毎月可視化 |
| 周囲を巻き込む | 税理士に渡せるレポート出力 |

## 3. TCの構造的優位性

### 3.1 AIに渡せるデータの質(v1.2から継承、感情選択式化対応)

| データソース | Supabaseテーブル | AI活用ポイント |
|------------|----------------|--------------|
| トレード記録 | trades(30カラム) | 通貨ペア、方向、ロット、エントリー/決済価格、損益、時刻、タグ |
| チェックリスト | trades.checklist | ルール遵守の構造化データ(YES/NO/PARTIAL) |
| 振り返り | trades.reflection | 自由テキスト+構造化評価 |
| 手法 | user_settings.methods | 手法別の分類基準 |
| リスク管理 | trades.risk_status等 | 許容損失対実損失の比較 |
| 経費 | expenses | カテゴリ別・日付別の経費データ |
| 入出金 | capital_records | 入出金履歴と口座残高推移 |
| 相場ノート | notes | 日付別の相場観・メモ |
| 感情記録 | trades.entry_emotion | 8カテゴリ構造化(選択式化対応済み) |

### 3.2 競合に真似できない4つの壁(v1.3で1つ追加)

1. **経費管理統合**: 日本の全競合ツールで経費管理を持つものはゼロ
2. **エントリー前チェックリスト**: ブローカーが構造的に真似できない
3. **日本税制への最適化**: 国内FX/海外FX、20万円ライン、住民税普通徴収対応
4. **【v1.3追加】1年サイクルでの業務OS化**: 記録→分析→申告→翌年戦略の循環構造。データの不可逆的蓄積。

## 4. 設計原則

### 4.1 v1.2から継承する原則

| 原則 | 説明 |
|------|------|
| 記録が先、AIは後 | TCの哲学を守る |
| 具体的であること | 「がんばりましょう」ではなく数値根拠 |
| 日本語ネイティブ | 日本の会社員トレーダーの文脈 |
| データ主権 | Anthropicの利用規約に基づきAPI送信データは学習に使用されない |
| フェイルセーフ | API障害時はAI機能のみ停止、基本機能は継続 |

### 4.2 v1.3 で新規追加する原則

| 原則 | 説明 |
|------|------|
| **不景気耐性** | 「失うコスト > 乗り換えコスト」を構造で作る |
| **自己投資の可視化** | 毎月「節税額・回避損失・時間節約」を金額で見せる |
| **シンプルさが正義** | 機能数の足し算でなく、体験のシンプルさを優先 |
| **3層構造UI** | 表(主役)/奥(脇役)/裏方(自動)で機能を整理 |
| **データ駆動改善** | リリース前ヒアリングではなく、リリース後の行動データで改善 |
| **3条件遵守** | 受動性・行動可能性・金額換算を全機能で守る |

### 4.3 シンプルさの2レベル(v1.3 追加)

```
レベル1: 体験のシンプルさ(必須)
  → 使い始めて5分で価値が分かる、画面が散らからない
  → 入口は2つ(週次レビュー / 確定申告AI)

レベル2: 機能数のシンプルさ(間違い)
  → 「Premium機能が少ない」と認識されてはダメ
  → 奥には豊富な機能を用意する
```

価格差 ¥1,000(Pro→Premium)を正当化するには、機能数も十分に必要。
ただし**入口はシンプル**に、**奥は豊富**に、の構造を守る。

## 6. Phase 1: 確定申告AI

### 6.1 概要(v1.3 で主役化)

| 項目 | 内容 |
|------|------|
| 機能名 | 確定申告AIアシスタント |
| 目的 | 副業トレーダーの確定申告ストレスを年1回の救世主体験に変える |
| **位置づけ** | **Premium の主役・参入障壁の中核** |
| リリース | **2026年9月**(v1.2では12月だったが前倒し) |
| 使用モデル | Claude Sonnet 4.6 |
| 想定使用頻度 | 11月〜2月にピーク、年1回必ず発動 |
| 最低データ条件 | 1ヶ月以上の取引データ |

### 6.2 主役化の根拠

リサーチ結果と議論で確認された3点:

1. **日本特化の参入障壁**: TraderSync, TradeZella, Edgewonk, TradesViz等の英語圏SaaSが構造的に手を出せない領域。日本の税制は複雑で、ローカライズコストが高い。
2. **「ないと困る」の3条件を完全充足**:
   - 受動性: 11月になると自動でリマインド
   - 行動可能性: 経費分類・節税額が即座に見える
   - 金額換算: 「節税額」が金額そのもの
3. **不景気耐性の核**: 確定申告という業務は景気に関係なく毎年発動する。「今年の損失繰越」「来年の節税」というデータが TC にしかない状態を作れば、絶対に切れない。

### 6.3 サブ機能

#### A. 経費自動分類提案

ユーザーの経費入力に対して、AI が自動でカテゴリと勘定科目を提案。

**入力データ:**
- 経費の名称、金額、日付、メモ
- ユーザーの過去の経費分類履歴(学習データ)

**出力:**
- 推奨カテゴリ(通信費、書籍代、セミナー代、PC代、VPS代等)
- 経費として認められる可能性(高/中/低)
- 認められない可能性が高い場合はその理由

**例:**
```
入力: 「Netflix」¥1,490
出力: 経費として計上推奨度【低】
     理由: 一般的な娯楽サービスのため、業務関連性の説明が必要
     代替案: トレード関連の動画講座を主に視聴している場合は
            「研修費」として計上可能性あり(税務署判断による)
```

#### B. 年間所得リアルタイム監視

副業トレーダー特有の「年20万円ライン」「住民税の壁」を継続監視。

**監視項目:**
- 年間累計利益(国内FX分・海外FX分を分離)
- 年20万円ラインの超過予測(2-3ヶ月先の推定)
- 海外FX(総合課税)の所得税率変動シミュレーション

**通知タイミング:**
- 年20万円超過確実時(住民税申告必要のリマインド)
- 海外FXで税率階段が上がる前(195万円、330万円ライン等)

#### C. 節税額の可視化

「TCを使うことで、あなたが今年守ったお金」を毎月表示。

**計算ロジック:**
```
節税額 = (経費未計上だった額の累計 × 税率) +
        (損益通算で取り戻した税額) +
        (損失繰越控除の保全価値)
```

**表示例:**
```
📊 2026年4月の節税効果

今月の経費計上額    ¥18,400
└ うち、AI提案で気づいた経費 ¥6,200

累計節税額(2026年)  ¥48,200
└ 経費漏れ防止: ¥31,200
└ 損益通算最適化: ¥17,000
```

#### D. 確定申告サポートレポート

11月〜2月に毎月配信される、確定申告準備の進捗レポート。

**コンテンツ:**
- 必要書類のチェックリスト(済/未済)
- 年間収支サマリー(申告書添付用フォーマット)
- 経費明細CSV(税理士提出用)
- e-Tax用XMLエクスポート(将来検討)
- 副業バレ対策(住民税普通徴収の選び方ガイド)

└── 申告書出力(税理士提出用)

既存タブ:分析タブ
└── 「節税額カード」追加(ホーム画面でも常時可視)
```

### 6.5 法的留意事項(極めて重要)

**TC は税務代理を行わない。** 必ず以下を全画面に明示:

> ⚠️ TCは税務に関する情報提供ツールであり、税務代理・税務相談を行うものではありません。
> 提案内容は参考情報です。実際の申告にあたっては、税理士または税務署にご相談ください。

税理士法 第52条「税理士業務の制限」抵触リスクを徹底回避するため、**「具体的な税額計算」「個別の税務相談」「申告書の作成代行」は行わない**。あくまで「データ整理」「カテゴリ提案」「金額の自動集計」までに留める。

### 8.2 設計思想

NetflixやSpotifyは典型的なコスト(消える)。
freeeやマネフォは投資寄り(節税額が残る)。

TCはこの「投資効果」を毎月可視化することで、月¥3,000を「安い」と感じさせる。

### 8.3 4つの数値

毎月のダッシュボードに以下4つを表示:

```

1. 節税額(年間累計) ¥48,200
   └ 経費漏れ防止 + 損益通算最適化

2. 時間節約        12時間28分
   └ Excelで同じ作業をする場合との比較

3. 回避損失見込み  ¥23,400
   └ What-Ifシミュレーションから算出

4. プレイブック資産 14件
   └ AI生成の「あなたの勝ちパターン」

──────────────────────
合計効果       ¥71,600 + 12時間28分
今月の利用料    ¥2,980
ROI            約24倍
```

### 8.4 計算ロジック

#### 節税額
```
節税額 = (経費計上額 × 限界税率) +
        (損益通算による節税額) +
        (損失繰越控除の保全価値)
```

#### 時間節約
```
時間節約 = (記録の自動化分) +
        (分析の自動化分) +
        (申告書作成の自動化分)
```
各項目は固定値ベース(将来は実測)。
- 記録の自動化: 1取引あたり3分
- 分析の自動化: 1月あたり2時間
- 申告書作成: 年1回で20時間

#### 回避損失見込み
```
回避損失見込み = What-Ifシミュレーションの「もし守っていたら」値の累計
```

#### プレイブック資産
```
AI が生成した「あなたの勝ちパターン」の累計件数
```

## 10. 文章スタイルガイドライン

### 10.1 全体方針

TC のAIコーチ(Phase 2)は Claude Sonnet 4.6 を使用。
以下のスタイル原則をシステムプロンプトで明示し、出力品質を一定化する。

### 10.2 一人称・呼称

- AI側: 「私は」「TCコーチ」と名乗らない(透明な存在)、主語を伏せる
- ユーザー呼称: 「あなた」「あなたのトレード」「あなたのデータ」
- 敬語: です・ます調(日本人30-40代会社員ターゲット)
- トーン: 砕けすぎない、上から目線でもない

### 10.3 1回のレビューで触れる改善点の数

- **改善点(否定的指摘): 必ず1つに絞る**(多くて2つ)
- **肯定的フィードバック: 3つ**(Losada比 3:1〜5:1 の遵守)
- 多すぎると行動変容が起きない
- 少なすぎると価値を感じない

### 10.4 改善提案の構造(5段構成・必須)

```
【1. 現状の事実】
  「先週、あなたは13回エントリーし、うち4回が金曜午後でした」

【2. 数値根拠】
  「金曜午後のトレード4回中3回が損失。損失合計は -8,400円。
  月換算で約 -33,600円のペース」

【3. 行動の解釈(CBT的)】
  「あなたのプレイブックでは『金曜午後はボラ低下のため避ける』と
  されています。今週はこのルールから外れたエントリーが3回ありました」

【4. 改善提案(Tiny Habit)】
  「次回の金曜13時、チャートを開く前に **3秒だけ** プレイブックの
  『金曜の項目』を読み直してみてください。それだけで十分です」

【5. 期待効果(金額換算)】
  「このパターンを直すと、来月の収支は試算で +25,000円〜+33,000円
  改善する可能性があります」
```

### 10.5 損失指摘の言い回し(NG/OK)

| NG | OK | 理由 |
|----|----|------|
| 「あなたはルールを守れていません」 | 「先週、3回ルール外のエントリーがありました」 | 人格でなく行動 |
| 「リベンジトレードの傾向があります」 | 「先週、損失直後30分以内のエントリーが2回あり、両方とも損失でした」 | 行動と結果の事実 |
| 「損切りを早めるべきです」 | 「先週、損切り平均が1.4Rでした。1Rルール通りなら、約4,500円の損失を回避できた可能性があります」 | 数値+金額換算 |
| 「金曜のトレードはやめましょう」 | 「過去90日のデータでは、金曜のあなたの勝率は34%です(他曜日平均58%)。**金曜だけ避けると、年換算で +120,000円の改善見込み**です」 | 期待効果の金額提示 |
| 「メンタルが弱いですね」 | 「先週、感情スコアを『焦り 7/10』とご自身でつけたトレードが2回あり、そのうち2回とも損失でした」 | 自己申告データの引用 |

### 10.6 文章サンプル(週次レビュー出力例)

```

## 17. リリース後の検証体制

### 17.1 思想

**v1.2では「リリース前にヒアリング50人」を想定していたが、v1.3 ではこれを廃止。**

理由:
1. ヒアリング結果は自己選択バイアスがあり、「市場の真実」ではない
2. 人は「何が欲しいか」を正確に答えられない(Henry Ford 問題)
3. SaaSの強みは「リリース後の継続改善」
4. 行動データはアンケートより遥かに正確

代わりに、リリース後の検証体制を強化する。

### 17.2 リリース後すぐに計測する4つの指標

```
1. Pro→Premium 転換率
   想定15% → 実際何%か?
   
2. Premium 加入者の機能使用率
   - 週次レビュー閲読率
   - 確定申告AI 利用率
   - 投資効果ダッシュボード閲覧率
   - What-Ifシミュレーション利用率
   - サジェスト質問クリック率
   
3. 月次チャーン率
   8%想定 → 実際何%か?
   
4. 解約理由の分布
   - 解約画面でのアンケート回答
   - 「機能不足」「価格」「使いこなせない」「もう使わない」
```

### 17.3 アンケート設計(リリース後限定)

#### A. 3ヶ月継続者アンケート

Premium 加入から3ヶ月後に自動配信(任意回答)。

```
1. TC Premium で一番役に立った機能は?(択一)
2. もう少し改善してほしい機能は?(自由)
3. 月¥2,980 の価値を感じていますか?(5段階)
4. 友人にTCを勧めたいですか?(NPS、0-10)
5. (NPS 9-10の人) 何が良かったですか?(自由)
6. (NPS 0-6の人) どこが不満ですか?(自由)
```

#### B. 解約者アンケート

解約画面で必ず表示(任意回答だが、「継続する」ボタンの隣に「アンケートに答えて解約」ボタンを設置)。

```
1. なぜ解約しますか?(複数選択)
   □ 機能不足
   □ 価格が高い
   □ FXをやめた
   □ 他のツールに乗り換える
   □ 使いこなせない
   □ その他
2. もしTCに何があれば継続しましたか?(自由)
3. 戻ってきたいと思いますか?(YES/NO)
```

### 17.4 ヘビーユーザーインタビュー

利用ログから以下を抽出し、月1回 Zoom 30分インタビューを依頼:

- 週次レビュー閲読率 95% 超
- 月内利用日数 20日以上
- 加入から6ヶ月以上継続

これは「実際に使い込んでいる人の声」で、最も価値が高い。
ヒアリング負荷を下げるため、月1回・1人ずつで十分。
