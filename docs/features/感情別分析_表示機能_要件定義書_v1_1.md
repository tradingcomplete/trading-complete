## 1. 概要

### 1.1 背景

感情記録が選択式（8種類の単一選択）に変更されたことで、構造化データが蓄積される。
このデータを3箇所で活用する:
1. 分析タブの感情別パフォーマンス分析（アコーディオン）
2. トレード記録タブの感情バッジ表示
3. エントリー時の感情必須バリデーション（データ品質の保証）

### 1.2 MODULES.md 準拠の参照先

MODULES.md で定義済みの感情関連仕様:

| 参照先 | MODULES.md 記載 |
|--------|----------------|
| 定数名 | `EMOTION_OPTIONS`（8感情カテゴリ定数） |
| 正規化関数 | `normalizeEmotion(raw)` → `{ selection, memo }` |
| UI操作関数 | `toggleEmotion(btn)`, `getEmotionFromSelector()`, `setEmotionToSelector()` |
| 定義ファイル | TradeEntry.js の先頭で定義、window に公開 |
| AISummaryModule出力 | `emotionStats: { byEmotion, positiveTotal, negativeTotal, untagged }` |

## 2. 機能詳細

### 2.1 感情選択の必須化（TradeEntry.js / TradeEdit.js）

感情データなしのトレードが記録されると、以下の分析機能が崩れるため、バリデーションで防ぐ。

#### TradeEntry.js — 新規エントリー時

`#validateTradeData(data)` に感情チェックを追加:

```
既存のバリデーション順序:
  1. 通貨ペア必須チェック
  2. 売買方向必須チェック
  3. エントリー日時必須チェック
  4. 価格バリデーション
  5. チェックリストバリデーション（3つの根拠）
  6. ★ 感情選択必須チェック（追加）

感情チェック:
  - formData.entryEmotion を getEmotionFromSelector() で取得（L405）
  - selection が空文字 → alert + return false
  - メッセージ: 「エントリー時の感情を選択してください」
```

#### TradeEdit.js — 編集保存時

現在の動作（L129-135）:
```javascript
// 未選択＋メモなし → 元データを保持（暗黙のフォールバック）
if (!emotionData.selection && !emotionData.memo) {
    return trade.entryEmotion;
}
```

問題: 元データが旧形式（selection 空）の場合、空のまま保存される。

対策: `#validateUpdateData()` に感情チェックを追加:
```
- 編集後の entryEmotion を確認
- selection が空の場合 → alert + return false
- メッセージ: 「エントリー時の感情を選択してください」
```

### 2.2 分析タブ — 感情別分析アコーディオン（ReportModule.js）

#### 配置場所

```
▼ 通貨ペア / 商品分析
▼ 曜日別分析（エントリー日時ベース）
▼ セッション別分析
▼ トレード履歴
▼ ルール遵守・リスク分析
▼ 😌 感情別分析            ← ★ ここに追加
▼ 振り返り一覧
```

#### 表示内容

アコーディオンを開くと、2つのテーブルが表示される:

**テーブル1: ポジティブ vs ネガティブ サマリー（2行）**

| 分類 | 件数 | 勝敗 | 勝率 | Pips | 期待値(p) | 期待値(¥) |
|------|------|------|------|------|-----------|-----------|
| 😌 ポジティブ | 15件 | 10勝5敗 | 66.7% | +120.5 | +8.0p | +¥5,000 |
| ⚡ ネガティブ | 8件 | 2勝6敗 | 25.0% | -85.3 | -10.7p | -¥3,200 |

**テーブル2: 感情8種類 詳細（データありの感情のみ表示）**

| 感情 | 件数 | 勝敗 | 勝率 | Pips | 期待値(p) | 期待値(¥) |
|------|------|------|------|------|-----------|-----------|
| 😌 冷静 | 5件 | 4勝1敗 | 80.0% | +65.2 | +13.0p | +¥3,500 |
| 💪 自信あり | 4件 | 3勝1敗 | 75.0% | +32.0 | +8.0p | +¥2,100 |
| ... | ... | ... | ... | ... | ... | ... |

仕様:
- Pips の +/- に応じて色分け（`#00ff88` / `#ff4466`）— 既存テーブルと同じ
- データ 0 件の感情は行を非表示
- 全感情 0 件の場合は「感情データがありません」メッセージ
- selection が空文字のトレードは集計スキップ

#### 集計ロジック（AISummaryModule の emotionStats パターンに準拠）

```
データ構造（MODULES.md L240 のパターンに合わせる）:
  byEmotion: {
      calm:      { count, wins, losses, pips, yen, yenCount },
      confident: { count, wins, losses, pips, yen, yenCount },
      ... 8感情分
  },
  positiveTotal: { count, wins, losses, pips, yen, yenCount },
  negativeTotal: { count, wins, losses, pips, yen, yenCount }

集計フロー:
  1. window.EMOTION_OPTIONS を参照
  2. 各トレードの entryEmotion を window.normalizeEmotion(raw) で正規化
  3. selection が空のトレードはスキップ
  4. EMOTION_OPTIONS から category (positive/negative) を判定
  5. byEmotion[selection] と positiveTotal/negativeTotal に加算
```

```

  2. 感情別統計オブジェクトを初期化（byEmotion + positiveTotal + negativeTotal）

     - 全0件の場合は「感情データがありません」

  テーブルカラム・色分けは #generateRuleRiskAnalysis の generateRow() と同じパターン
```
