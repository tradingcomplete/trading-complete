## アフィリエイトとは？

### 基本の仕組み

```
┌──────────┐      ┌──────────┐      ┌──────────┐
│ あなた   │ 紹介 │ ユーザー │ 登録 │ブローカー│
│Trading   │─────→│ (高市さん)│─────→│ (DMM FX) │
│Complete  │      │          │      │          │
└──────────┘      └──────────┘      └──────────┘
                                          │
                                          │ 紹介料支払い
                                          ↓
                                    ┌──────────┐
                                    │ あなた   │
                                    │(報酬獲得)│
                                    └──────────┘
```

**簡単に言うと：**
「いいブローカーを紹介したら、お礼をもらえる仕組み」

### 2つのタイプ

#### タイプ1：CPA（Cost Per Action）

```
【仕組み】
ユーザーが口座開設 → 1回だけ報酬

【報酬例】
- 日本のブローカー：5,000円〜20,000円/人
- 海外のブローカー：10,000円〜50,000円/人

【計算例】
月10人が口座開設（平均10,000円）
= 月100,000円の収益
```

#### タイプ2：IB（Introducing Broker / 継続報酬型）

```
【仕組み】
紹介したユーザーが取引するたび → 継続的に報酬

【報酬例】
- 1ロット（10万通貨）取引 → 200円〜500円
- ユーザーが取引し続ける限り報酬継続

【計算例】
10人のユーザーが月平均50ロット取引
= 10人 × 50ロット × 300円 = 月150,000円

【メリット】
- 継続収益（ストック収益）
- ユーザーが増えるほど収益増
- 長期的に安定
```

### Trading Completeでの実装方法

```
【シンプルな実装】

1. 設定画面に「ブローカー選択」欄を追加
   ├─ DMM FX
   ├─ GMOクリック証券
   ├─ 外為どっとコム
   └─ その他（自分で入力）

2. 各ブローカーの隣に「詳細・登録」ボタン
   → クリック → アフィリエイトリンク

3. 必ず明記する
   「※このリンク経由で登録いただくと、
    Trading Complete開発の支援になります。
    手数料などは一切変わりません。」

【透明性を保つ】
- すべてのページに開示
- ユーザーに追加コストなし
- 信頼できるブローカーのみ紹介
```

### アフィリエイトのメリット・デメリット

**メリット：**
- ✅ ユーザーは無料でブローカー口座開設
- ✅ あなたは追加収益を得られる
- ✅ システム開発費用の一部を賄える
- ✅ ブローカーにとっても新規顧客獲得

**デメリット・注意点：**
- ⚠️ 利益相反のリスク（報酬目的で推奨してると思われる）
- ⚠️ 透明性が必須（隠すと信頼失墜）
- ⚠️ 信頼できるブローカーのみ選ぶ

**対策：**
- 明確な開示
- ユーザー利益を最優先
- 複数のブローカーを提示（選択肢を提供）
- 非アフィリエイトブローカーも掲載

## AI分析機能の注意点

### グレーゾーン回避のルール

**Trading Completeは投資助言を行わない記録・分析ツールです。**

#### ❌ 絶対に使ってはいけない表現

```
【投資助言と見なされる表現】
× 「今エントリーすべき」
× 「このトレードを推奨します」
× 「勝率90%のシグナル」
× 「確実に勝てる」
× 「このポジションを持つべき」
× 「利益確定のタイミングは今」

【問題点】
→ 具体的な売買指示に見える
→ 投資助言業の登録が必要になる可能性
→ 金融商品取引法違反のリスク
```

#### ✅ 使って良い表現（過去データの分析）

```
【記録・分析ツールとして適切な表現】
○ 「過去のデータでは、月曜日の勝率が60%でした」
○ 「あなたのトレードは東京時間に集中しています」
○ 「損切りが遅れる傾向が見られます」
○ 「このパターンは過去10回中7回勝っています」
○ 「平均保有時間は2.5時間です」
○ 「リスクリワード比は1:1.8です」

【OK な理由】
→ 過去の事実を述べているだけ
→ 未来の予測・推奨ではない
→ ユーザー自身の判断を尊重
```

### AI機能の実装ガイドライン

```
【Premiumプランの AI分析機能】

1. パターン認識
   ○ 「過去にこのようなチャートパターンが出現した時、
      あなたのトレード成績は〇〇でした」
   × 「今エントリーすべきパターンです」

2. リスク分析
   ○ 「あなたの平均損失額は利益額の1.5倍です」
   ○ 「ロット数を2倍にした時の過去の成績は〇〇でした」
   × 「ロット数を減らすべきです」

3. 時間帯分析
   ○ 「東京時間の勝率：45%、ロンドン時間：65%」
   ○ 「金曜日のトレード回数が最も多く、勝率は低めです」
   × 「金曜日はトレードしない方がいい」

4. 感情分析（将来機能）
   ○ 「連敗後のトレードは勝率が20%低下しています」
   ○ 「大きく勝った翌日は損失が大きい傾向があります」
   × 「連敗したらトレードを止めるべき」
```

```

✓ 失敗談
```

```
収益 < コミュニティの信頼

○ 長期的信頼のために質を保つ

× アフィリエイト報酬目的の推奨
○ ユーザー利益を最優先

× 要望を無視して開発
○ 要望を最優先で実装
```

### 4. 継続を仕組み化

```
【エンゲージメント機能】

→ 記録を習慣化させる
```

```
感覚ではなく数字で判断：

```

```
計画：保守的な数値で
実行：全力で取り組む

× 楽観的な計画で失敗
○ 保守的な計画で余裕を持つ

× 悲観的で行動しない
○ 楽観的に全力で実行
```
