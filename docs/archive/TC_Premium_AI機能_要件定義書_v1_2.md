## 1. エグゼクティブサマリー

### 1.1 なぜ今、AI機能が必要か

2026年3月時点で、英語圏のトレードジャーナル市場ではAI機能が「あって当然」の標準装備に移行した。TraderSyncの「Cypher AI」は月額$29.95〜$79.95の全プランでAI機能を提供し、TradeZellaの「Zella AI」は50以上の分析レポートを即座に生成する。

一方、日本語対応のAI搭載FXトレード記録ツールは2026年3月時点で確認できない。これは脅威ではなく、TCにとっての明確な「先行者利益の窓」である。

### 1.2 TCのAI機能コンセプト

> **「記録が資産になる瞬間」を作る**

TCの哲学は「記録してからトレード」。AIはその記録を「蓄積するだけのデータ」から「あなた専用の成長エンジン」に変える。

汎用AI（Claude Cowork、ChatGPT等）との根本的な違いは、TCが持つ「構造化された高品質データ」にある。同じAIモデルでも、入力データの品質が出力品質を決定的に左右する。これはTCだけが提供できる価値だ。

### 1.3 一文まとめ

> 「あなたのトレード記録をAIが読み解き、具体的な改善点を教えてくれる」
> それが Premium の約束。

## 2. 第三者市場検証

### 2.1 英語圏トレードジャーナルのAI搭載状況（2026年3月調査）

| ツール | AI機能名 | 主な機能 | 価格帯（月額） |
|--------|---------|---------|--------------|
| TraderSync | Cypher AI / Cypher Coach | パターン自動検出、会話型Q&A、プロアクティブ監視（Elite） | $29.95〜$79.95 |
| TradeZella | Zella AI | 50+分析レポート自動生成、トレーダー教育統合 | $24〜$33 |
| TradesViz | 自然言語Q&A | 「先月のGBPJPYの勝率は？」→チャート付き回答 | 無料プランでも利用可 |
| Edgewonk | Edge Finder / Tiltmeter | 週次自動分析、感情的偏り検出 | $169〜$197/年 |

**市場の鍵となる知見:**

TraderSyncのCypherが市場をリードしている。100トレード以上蓄積すると自動的にAI分析が開始され、プロアクティブな監視機能（Cypher Coach）は最上位プランのElite（$79.95/月）でのみ利用可能。2026年のレビューでは、多くのレビュアーがAI機能をTraderSyncの最大の差別化要素として評価している。

TradeZellaは「教育」を軸にした差別化を推進しており、Zella Universityという学習プログラムとAI分析を組み合わせている。

### 2.2 日本市場の空白（2026年3月確認）

日本で利用可能なFXトレード記録ツールのAI搭載状況を第三者的に調査した結果:

| ツール | AI機能 | 経費管理 | 確定申告支援 |
|--------|--------|---------|------------|
| Myfxbook | なし | なし | なし |
| FX Trade Analytics | なし | なし | なし |
| TradeNote（アプリ） | なし | なし | なし |
| トレードメモ（アプリ） | なし | なし | なし |
| 日々トレFX | なし | なし | なし |
| FOREX.com 行動科学分析 | あり（プラットフォーム限定） | なし | なし |
| FX業者内蔵ツール（みんなのFX等） | AI予測のみ（記録分析ではない） | なし | なし |

**結論**: 日本市場において「ユーザー自身のトレード記録をAIで分析する」ツールは皆無。FX業者が提供するAI機能は「為替予測」（値動き予想）に特化しており、「自分のトレードの改善」を目的としたAIは存在しない。ここにTCの巨大な機会がある。

### 2.3 汎用AIの代替可能性の再評価

初期リサーチの結論を第三者的に検証した結果、以下が確認された:

**Claude Cowork**: セッション非永続性という構造的限界は2026年3月時点でも変わっていない。Projects機能が追加されたが、継続的なデータ蓄積・構造化は設計思想の範囲外。Anthropic公式が金融文書へのアクセスに対して注意喚起を継続している点も重要。

**Notion AI**: GPT-5やClaude等の複数モデル切替が可能になったが、500件以上のレコードでの遅延問題は構造的に解決されていない。FXトレード分析に必要な「通貨ペア別×時間帯別×手法別」のクロス集計はNotion AI単体では困難。

**結論**: 汎用AIがTC的な専門ツールを代替するリスクは低い。ただし、英語圏のAI搭載ジャーナル（特にTraderSync）が日本語対応する可能性は中期的な脅威として残る。LLMの多言語能力は急速に向上しており、先行実装が最大の防御策。

## 3. TCの構造的優位性

### 3.1 AIに渡せるデータの質

TCは以下の構造化データを保持しており、これがAI分析の品質を決定的に高める:

| データソース | Supabaseテーブル | AI活用ポイント |
|------------|----------------|--------------|
| トレード記録 | trades（30カラム） | 通貨ペア、方向、ロット、エントリー/決済価格、損益、時刻、タグ |
| チェックリスト | trades.checklist | ルール遵守の構造化データ（YES/NO/PARTIAL） |
| 振り返り | trades.reflection | 自由テキスト＋構造化評価 |
| 手法 | user_settings.methods | 手法別の分類基準 |
| リスク管理 | trades.risk_status等 | 許容損失対実損失の比較 |
| 経費 | expenses | カテゴリ別・日付別の経費データ |
| 入出金 | capital_records | 入出金履歴と口座残高推移 |
| 相場ノート | notes | 日付別の相場観・メモ |
| 感情記録 | trades.entry_emotion | エントリー時のメンタル状態（**選択式化 実装中**: 8カテゴリ構造化データ → 感情別勝率の自動算出が可能に） |

### 3.2 AISummaryModuleの既存基盤

AISummaryModule v1.0.0 が既に以下の集計機能を実装済み:

```
generateSummary(periodType, year, month)
├── basic: 総トレード数、勝率、総Pips、円建て損益、PF
├── ruleCompliance: 遵守率、遵守時/非遵守時の成績
├── riskManagement: 許容内率、ステータス別成績
├── methodStats: 手法別の成績
└── emotionStats: 感情別の成績（※感情選択式化の実装後に追加予定）
```

これはAI機能の「データ準備層」として完成しており、Phase 1実装を大幅に加速できる。
感情選択式化（実装中）により `emotionStats` が追加され、AIの分析精度がさらに向上する。

### 3.3 競合に真似できない3つの壁

1. **経費管理統合**: 日本の全競合ツールで経費管理を持つものはゼロ。AI×経費データ＝確定申告支援は唯一無二。
2. **エントリー前チェックリスト**: 「記録してからトレード」の強制力。ルール遵守データが構造化されている。ブローカーがこの仕組みを真似するインセンティブは構造的に存在しない（ブローカーはトレード回数が増えるほど利益が出る）。
3. **日本税制への最適化**: 国内FX（申告分離課税20.315%）と海外FX（総合課税・累進）の自動判定、年間所得20万円ラインの監視。

### 4.2 設計原則

| 原則 | 説明 |
|------|------|
| **記録が先、AIは後** | TCの哲学を守る。AIはユーザーの代わりに記録しない。記録した結果を分析する |
| **具体的であること** | 「がんばりましょう」ではなく「22時以降のトレードを控えると月間損益が推定15%改善」 |
| **日本語ネイティブ** | 英語翻訳ではなく、日本の会社員トレーダーの文脈で書かれた提案 |
| **データ主権** | ユーザーのデータはAI分析のためだけに使用。第三者共有なし。学習データにもしない |
| **フェイルセーフ** | API障害時はAI機能のみ停止。記録・分析の基本機能は影響を受けない |

## 5. 機能詳細 - Phase 1: AIトレードコーチ

### 5.1 概要

| 項目 | 内容 |
|------|------|
| 機能名 | AIトレードコーチ |
| 目的 | ユーザーのトレード記録から改善点を発見し、具体的な提案を提供 |
| リリース | 2026年9月（Premium ローンチと同時） |
| 使用モデル | Claude Sonnet 4.6 |
| 使用量制限 | 月30回まで（1日1回の想定 + 週末分析余裕） |
| 最低データ条件 | closedトレード20件以上（精度確保のため） |

### 5.2 3つのサブ機能

#### A. 月次パフォーマンスレビュー

毎月自動生成されるパーソナライズドレポート。AISummaryModuleの出力をClaude APIに送信し、自然言語のレビューを生成する。

**入力データ（AISummaryModule.generateTextSummary()の出力を活用）:**

```
【2026年8月 トレードサマリー】
- 総トレード数: 45件
- 勝率: 58% (26勝19敗)
- 総Pips: +320pips
- 円建て総損益: +¥48,000
- PF: 1.42
- ルール遵守率: 71%（遵守時勝率72%、非遵守時38%）
- 許容損失内率: 82%
- 手法別: ブレイクアウト 65%(20件), 押し目買い 48%(15件)
- 時間帯: 東京 72%, ロンドン 55%, NY 41%
- 曜日: 月〜水 62%, 木金 45%
- 感情記録: 「冷静」時勝率68%、「焦り」時32%
```

**出力例（AIが生成）:**

```
📊 8月のトレードレビュー

先月のトレード、お疲れさまでした。
全体としては+¥48,000のプラスで、着実に成果が出ていますね。

特に良かった点:
東京セッション（9:00〜15:00）での勝率72%は非常に安定しています。
ブレイクアウト手法の勝率65%も、先月の58%から改善しました。

改善できそうな点:
1. NY時間（22時以降）の勝率が41%と低めです。
   45件中12件がNY時間帯ですが、この12件を除いた場合の
   総損益は+¥62,000になります。
   → 「22時以降はトレードしない」だけで月¥14,000の改善見込み。

2. 木・金曜の勝率が45%に落ちています。
   週後半に疲れがたまっている可能性があります。
   → 金曜日はエントリーを1件に制限してみてはいかがでしょうか。

3. ルール非遵守時の勝率38%は危険信号です。
   13件のルール非遵守トレードで¥22,000の損失が出ています。
   → チェックリストの徹底だけで損失の大部分を回避できます。
```

#### B. トレードパターン分析

個別トレード群から「勝ちパターン」「負けパターン」を自動抽出する。

**分析軸:**

| 分析軸 | データソース | 抽出パターン例 |
|--------|------------|-------------|
| 時間帯 | trades.entry_time | 「東京オープン直後（9:00-9:30）は勝率が特に高い」 |
| 曜日 | trades.entry_date | 「月曜のトレードは他曜日より勝率が15%高い」 |
| 通貨ペア | trades.symbol | 「ドル円は安定だがポンド円は大きく負けている」 |
| 手法 | trades.method_id | 「ブレイクアウトは勝率が高いが、押し目買いのPFの方が良い」 |
| ルール遵守 | trades.checklist | 「チェックリスト完全遵守時のPFは2.1で非遵守時の3倍」 |
| リスク管理 | trades.risk_status | 「許容損失内のトレードは勝率が22%高い」 |
| 感情 | trades.entry_emotion（選択式） | 「冷静時の勝率73%、焦り時20%」「ポジティブ感情時PF 2.1、ネガティブ時0.6」 |
| 保有時間 | 決済日時-エントリー日時 | 「1時間以内の決済は勝率が低い（スキャルピングへの傾斜）」 |

#### C. 改善提案生成

パターン分析結果をもとに、3つの具体的アクションを提案する。

**提案の形式（固定フォーマット）:**

```
💡 今月の改善提案

提案1: [具体的なアクション]
根拠: [データに基づく説明]
期待効果: [定量的な改善見込み]

提案2: ...
提案3: ...
```

**制約:**
- 提案は必ずユーザー自身のデータに基づく（一般論禁止）
- 期待効果は必ず数値で示す（「良くなる」ではなく「+¥X,XXX」）
- 投資助言にあたる表現は禁止（「買うべき」「売るべき」は使わない）
- AIの限界を明示（「これは過去データに基づく分析であり、将来の成績を保証するものではありません」）

### 5.3 AIプロンプト設計方針

```
【システムプロンプト骨格】

あなたはFXトレードの改善コーチです。
ユーザーは日本在住の30代会社員で、副業としてFXトレードをしています。

以下のルールを厳守してください:
1. 必ずユーザーのデータに基づいて話す。一般的なアドバイスは禁止。
2. 「〜すべきです」ではなく「〜してみてはいかがでしょうか」の提案型。
3. 数値は必ずソースを示す（「あなたの先月のデータでは〜」）。
4. 投資助言にあたる表現は絶対に使わない。
   NG: 「ドル円を買うべき」「ショートがおすすめ」
   OK: 「あなたのドル円ロングの勝率は72%で、他ペアより安定しています」
5. 損失やミスを責めるトーンは禁止。前向きな改善提案に徹する。
6. 日本語は自然で、専門用語を使いすぎない。
7. 文末の免責事項を必ず含める。

【ユーザーデータ】
{AISummaryModule.generateTextSummary() の出力}

【追加コンテキスト（必要に応じて）】
{直近10件のトレード詳細}
{経費サマリー}
{相場ノートの要約}
```

## 6. 機能詳細 - Phase 2: 確定申告AIアシスタント

### 6.1 概要

| 項目 | 内容 |
|------|------|
| 機能名 | 確定申告AIアシスタント |
| 目的 | 会社員トレーダーの確定申告準備を大幅に省力化 |
| リリース | 2026年12月（確定申告シーズン前） |
| 差別化 | 日本の全FXツールで経費管理を持つものはゼロ。唯一無二の機能 |

### 6.2 なぜこの機能が最重要か

30代会社員FXトレーダーのペインポイント調査結果:

| 痛みの深さ | ペインポイント | 既存の解決策 |
|-----------|-------------|------------|
| 🔴 最大 | 確定申告が難しい・面倒 | 税理士（高い）or 自力（ミスリスク） |
| 🟡 中 | 年間所得20万円のラインが分からない | Excelで自力管理（漏れやすい） |
| 🟡 中 | 経費の按分計算が分からない | 確定申告書の手引きを読む（難解） |
| 🟢 低 | トレードの改善方法が分からない | YouTube、書籍、ブログ |

確定申告の痛みはトレードスキルの痛みより深く、時期が限定されている（1〜3月）ため訴求力が極めて高い。「12月までに加入すれば来年の確定申告が楽になる」というマーケティングが可能。

### 6.3 サブ機能

#### A. 経費自動分類提案

| 脅威 | 対策 |

## 12. リスクと対策

### 12.1 技術リスク

| リスク | 影響度 | 対策 |
|--------|-------|------|
| Claude API障害 | 中 | フェイルセーフ: AI機能のみ停止、記録・分析は影響なし。ユーザーに「一時的に利用できません」表示 |
| APIレスポンス品質の変動 | 中 | プロンプトのバージョン管理、出力の構造化（JSON指定）、異常検出ロジック |
| Anthropic料金改定 | 低 | 現時点で97%の利益率があるため、2〜3倍の値上げにも耐えうる |
| Edge Function実行時間超過 | 低 | タイムアウト設定（30秒）、長い分析はバッチ処理に切り替え |

### 12.2 ビジネスリスク

| リスク | 影響度 | 対策 |
|--------|-------|------|
| TraderSync等の日本語対応 | 高 | Phase 1を最速で実装。確定申告支援は日本市場特化のため対抗困難 |
| ユーザーがAI出力を投資助言と誤解 | 高 | 免責事項の徹底表示、「参考情報」であることの明示、利用規約への明記 |
| Premium転換率が想定以下 | 中 | Proユーザーへの定期的なティーザー表示、体験版（月1回無料）の提供 |
| AI依存によるチェックリスト軽視 | 低 | AIはチェックリストの「結果」を分析するもの。記録プロセス自体は変更しない |

### 12.3 法的リスク

| リスク | 対策 |
|--------|------|
| 投資助言業に該当する可能性 | 具体的な売買の推奨は一切行わない。過去データの分析結果提示に留める |
| 確定申告支援の正確性 | 「参考情報」として提供。税理士・税務署への相談を推奨する文言を必ず表示 |
| 個人情報保護 | Claude APIに送信するのは集計データのみ。Anthropicの利用規約に基づきAPIデータは学習に使用されない |

**投資助言業に該当しないための3条件:**
1. 具体的な銘柄・タイミングの売買推奨を行わない
2. 個別の投資判断を行わない
3. 一般的な分析結果の提示に留める

TCのAI機能はユーザー自身の過去データの分析に限定されるため、上記3条件を満たす。ただし、リリース前に弁護士確認を推奨。

詳細は「感情記録_選択式化_要件定義書_v1_0.md」を参照。

**背景**: 現在の感情記録は自由記入式（textarea）のため、表記揺れが発生し集計・分析が不可能。
選択式（8カテゴリ、複数選択可 + 任意メモ）に変更することで、AIが感情別勝率を自動算出できるようになる。

**感情カテゴリ（8種）:**
ポジティブ: 😌冷静 / 💪自信あり / 🎯集中 / 🛡️慎重
ネガティブ: ⚡焦り / 😟不安 / 🔥興奮 / 🤔迷い

**データ形式変更:**
旧: `entryEmotion: "冷静に分析できていた"`（テキスト）
新: `entryEmotion: { selections: ["calm", "focused"], memo: "特に問題なし" }`（構造化）
後方互換性: `normalizeEmotion()` で新旧形式を自動判定
