## 💡 重要ルール：DRY原則（価格・制限値の管理）

### ⭐ 価格と制限値は「2つのマスターファイル」だけに書く

具体的な金額（¥2,480 等）や制限数（50件等）は、以下の **2ファイルだけ** に記載する。  
他のすべてのファイルは「[Proプラン月額]」「[無料枠上限]」のような **プレースホルダー変数** で参照する。

| マスターファイル | 管理する内容 |
|----------------|------------|
| ⭐ `Trading_Complete_中核オファー要件定義書_v1_0.md` | プラン内容・価格・機能制限の最終定義 |
| ⭐ `Stripe決済システム_要件定義書_ロードマップ_v1_5.md` | 決済実装上の価格・プランID |

**理由**: 価格変更時に1ファイルだけ修正すれば全体に反映される。複数ファイルに直接金額を書くと、変更漏れによる矛盾が発生する。

### NG例 → OK例

```
❌ NG: Pro: ¥2,480/月（マーケティング資料に直接記載）
✅ OK: Pro: [Proプラン月額]/月（変数で参照）

❌ NG: 無料プランは50件まで（機能一覧に直接記載）
✅ OK: 無料プランは[無料枠上限]件まで（変数で参照）
```
