## 実装方針サマリ

### 採用する動作

**平常時**: ハッシュタグ 0 個  
**重要イベント当日**: 1 個のみ付与(#FOMC / #日銀 / #雇用統計 等)

### 関数構成

```
generateDynamicHashtags_(text, postType, rates)
  ├── [新規] getImportantEventTag_(today)
  │     ├── getEconomicCalendar_('today') の結果をパース
  │     ├── ★重要マーク付きの行のみ抽出
  │     └── 該当タグ候補配列を返す(なしは [])
  │
  ├── [新規] determinePrimaryEventTag_(bodyText, eventTags)
  │     ├── 本文の言及回数を計測
  │     ├── 最多タグが2回以上 → 主題タグ採用
  │     └── それ以外 → 優先度テーブル順で決定
  │
  └── 旧ロジック(#FX固定、通貨ペア、テーマ検出、フォールバック)は全削除
```

### 優先度テーブル(v1.0)

```
FOMC > 日銀 > 雇用統計 > CPI > PPI > GDP > ECB > BOE > RBA > ISM > その他
```

### 対応イベントタグ(v1.0)

| タグ | 本文キーワード |
|---|---|
| #FOMC | FOMC, パウエル |
| #日銀 | 日銀, BOJ, 植田, 金融政策決定会合 |
| #雇用統計 | 雇用統計, NFP, 非農業部門 |
| #CPI | CPI, 消費者物価 |
| #PPI | PPI, 生産者物価 |
| #GDP | GDP |
| #ECB | ECB, ラガルド |
| #BOE | BOE, ベイリー, 英中銀 |
| #RBA | RBA, 豪準備銀行, 豪中銀 |
| #ISM | ISM |

## 実装対象ファイル

**postFinalizer.gs**(唯一の変更対象・現状371行 → 実装後 約436行)

### 変更内容

1. **新規関数追加** (+約115行):
   - `getImportantEventTag_(today)`
   - `determinePrimaryEventTag_(bodyText, eventTags)`

2. **既存 `generateDynamicHashtags_` の全面書き換え** (約-80行, +40行):
   - #FX 固定付与を削除
   - TC言及検出(#個人開発)を削除
   - 通貨ペア検出を削除
   - テーマ検出を削除
   - フォールバック(#為替 / #トレード)を削除
   - 3個上限・URL制限ロジックを削除
   - 新仕様(平常時0個・イベント時1個)に置き換え

3. **ヘルパー関数の整理**:
   - `countAnyTag_`: 保持(`determinePrimaryEventTag_` 内で使用)
   - `countPairTag_`: 削除可能(通貨ペア検出を廃止するため)

### 変更しないファイル

- `promptBuilder.gs`: 変更不要
- `promptPolicies.gs`: 「ハッシュタグは書くな(システムが自動付与)」の記述は現状維持
- `textCleanup.gs`: 既存のハッシュタグ除去ロジックは保持(誤付与防止)
- その他: 全て変更なし

## Compana さんの働き方・コンパナ哲学(遵守)

- 修正指示は必ずダウンロードリンク付きで提供する
- 投稿テキストやClaudeの出力でアスタリスク(`*`)は絶対に使わない(AI感が出るため禁止)
- GAS関数の実行を依頼する際は、必ずその関数がどのファイルにあるかを明記する
- 「今日はここまで」「次のセッションで」等の気遣いは不要
- 「次回やりますか?」と先送り提案をしない。問題を見つけたらその場で最後まで修正しきる
- 設計書・要件定義書は内容が変わるたびにバージョン番号を上げる(同じバージョンに後から大きな変更を追加しない)
- デバッグは必ず先。根本原因を確認してから修正する
- アスタリスクは投稿本文・設計書本文どちらでも禁止
