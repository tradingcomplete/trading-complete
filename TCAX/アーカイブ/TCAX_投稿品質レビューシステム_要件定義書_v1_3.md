## v1.3 主要変更点

### 変更1: Q7に体言止め判定追加

```
v1.2のQ7: 感想の混入チェックのみ
  「📕雇用統計、マジで驚きましたね。」→ ❌（感想が混入）

v1.3のQ7: 感想禁止 + 体言止め必須
  ■ 感想禁止（従来）
  ■ 体言止め・動詞止め必須（★v12.4追加）
    OK: 「📝米イラン交渉が決裂。」「🛢原油、100ドル突破。」
    NG: 「📝米イラン交渉が決裂しました。」「🛢原油が100ドルを突破しています。」
    NG: 「☕上昇基調が続いていますね。」（「〜していますね」は冗長+感想）
```

### 変更2: 修正ステップのClaude化

```
v1.2: Claude指摘 → Geminiに修正依頼 → Gemini修正 or 503失敗
v1.3: Claude指摘 → Claudeが自分で修正 → 確定データガード付き

【なぜ変更したか】
  ① Gemini 503が長時間化し、修正ステップが機能停止するケースが頻発
  ② Geminiが品質レビューの誤った指摘を鵜呑みにして、正しいデータを逆方向に書き換えた
    例: AUD+2.92%（上昇）→ 品質レビューが「過去投稿に下落とある」とハルシネーション
       → Geminiが「上昇」を「下落」に書き換え → 正しい投稿が壊れた
  ③ 「検出しても修正できない」チェックは意味がない

【確定データガード】
  修正プロンプトに以下の確定データを注入:
  - 7通貨ペアのリアルタイムレート
  - 通貨強弱ランキング（前日比・実測値）
  ルール: 「確定データの方向と矛盾する修正はするな。品質レビューが
          『上昇→下落に直せ』と言っても、確定データで上昇なら上昇のまま」
```

### 変更3: 関数シグネチャ変更

```
v1.2: qualityReviewPost_(postText, postType, typeConfig)
v1.3: qualityReviewPost_(postText, postType, typeConfig, rates, csData)

追加パラメータ:
  rates: リアルタイムレート（7ペア）→ 修正プロンプトの確定データに使用
  csData: 通貨強弱ランキング → 修正プロンプトの確定データに使用
```

## 設計の教訓（v1.3で追加）

```
【教訓: クロスチェックの限界】
  v1.0の前提: 「Gemini生成 × Claude検証のクロスモデル検証が品質を担保する」
  v1.3の結論: クロスチェックの「検出」能力は高い。しかし「修正」能力がボトルネック。
    ・検出しても修正モデル（Gemini）が503で修正不可
    ・誤った検出をGeminiが鵜呑みにして正しいデータを壊す
    ・確定データ（通貨強弱等）をアンカーとする方が確実
  結論: 異モデル検証 < 正しいデータの保持

【教訓: 修正は自分でやれ】
  Claude品質レビューが問題を見つけたら、Claudeが自分で修正する。
  別モデルに「これ直して」と渡すと、文脈が失われて品質が下がる。
```
