## 1. 目的

毎時取得しているレートデータを活用し、通貨強弱の「流れ」を蓄積。
勢い（モメンタム）と初動を重視した分析基盤を構築する。

効果:
- generatePost内の通貨強弱計算コスト削減（シート読み取りのみ）
- 「EURが3時間連続首位」「USDの弱さが加速」等のトレンド情報
- ★初動検出（2時間横ばいから突然動き始めた通貨を検出）
- ダウ理論によるペア別の中期トレンド判定
- コンパナのトレード判断にも活用可能

## 2. 通貨強弱シート設計

シート名: `通貨強弱`（自動作成）
更新頻度: 毎時（scheduledFetchRatesから呼び出し）
保持期間: 7日分（168行）。超過分は自動削除

### 列定義（24列）

```
A: 日時             yyyy/MM/dd HH:mm
B: EUR(%)           通貨強弱スコア（前日終値比）
C: USD(%)
D: JPY(%)
E: GBP(%)
F: AUD(%)
G: 最強             通貨コード（例: EUR）
H: 最弱             通貨コード（例: AUD）
I: 強弱差(%)        最強-最弱（ボラティリティ指標）
J: EUR勢い          ★初動↑↑急加速 / ↑加速 / →維持 / ↓減速 / ↓↓急減速
K: USD勢い
L: JPY勢い
M: GBP勢い
N: AUD勢い
O: 3h最強           直近3時間で最も多く最強だった通貨
P: 3hトレンド       独歩高 / 優位 / 混戦 / データ蓄積中
Q: 注目ペア         最強x最弱のペア（例: EUR/AUD）
R: USD/JPYダウ      上昇トレンド / 下降トレンド / レンジ / 転換の兆し
S: EUR/USDダウ
T: GBP/USDダウ
U: EUR/JPYダウ
V: GBP/JPYダウ
W: AUD/JPYダウ
X: AUD/USDダウ
```

## 3. 勢い判定（5段階 + 初動フラグ）

```
↑↑急加速     前時比 +0.15%超
↑加速        前時比 +0.05%超
→維持        ほぼ変化なし
↓減速        前時比 -0.05%超
↓↓急減速     前時比 -0.15%超

★初動: 直前2時間が「→維持」だった通貨が突然動き始めた場合に付与
  例: ★初動↑↑急加速（2時間横ばいから急に強くなった）
  トレードの初動を捉えるための最重要シグナル
```

---

## 4. ダウ理論判定

日次レートシート（既存OHLC）の直近5日間を分析:

```
上昇トレンド:     高値切り上げ60%以上 AND 安値切り上げ60%以上
下降トレンド:     高値切り下げ60%以上 AND 安値切り下げ60%以上
転換の兆し:      直近2日で高値の方向が反転
レンジ:          上記いずれにも該当しない
データ不足:      日次レートが3日未満
```

## 7. T-CAXとの適合性分析（なぜこの2つなのか）

### 7-1. 通貨強弱 + ダウ理論がT-CAXに最適な理由

```
【データ適合】
  通貨強弱: T-CAXが既に持っている「7ペアの毎時レート」だけで計算できる
  ダウ理論: T-CAXが既に持っている「日次OHLC」だけで判定できる
  → 新しいデータソース不要。既存の毎時トリガーに乗せるだけ

【投稿スタイル適合】
  コンパナの投稿は「なぜ動いているか」を語るファンダメンタルズ + マクロの流れ
  通貨強弱 = 「今どの通貨が買われているか」→ 投稿の主役選定に直結
  ダウ理論 = 「中期トレンドは継続か転換か」→ 投稿の仮説に直結
  → コンパナの軸である「流れを読む」にぴったり合う

【運用適合】
  毎時1行追加。計算はシンプル。GAS 6分制限に影響しない（+5秒程度）
  スプレッドシートに蓄積されるので、コンパナ自身のトレード判断にも使える
  将来的にTrading Completeのユーザーにも提供できる可能性がある機能
```

### 7-2. 他のインジケーターが合わない理由

```
RSI（相対力指数）
  必要: 14期間以上の終値（分足〜時間足レベルの粒度）
  T-CAX: 1時間に1回のデータ → 精度が出ない
  投稿: 「RSIが70を超えたので売り」はコンパナのスタイルではない

MACD（移動平均収束拡散）
  必要: 26期間以上の終値
  T-CAX: 同上。データ粒度が粗すぎる
  投稿: テクニカルシグナルの解説はT-CAXの役割ではない

ボリンジャーバンド
  必要: 20期間の終値 + 標準偏差計算
  T-CAX: 計算は可能だが、1時間足では精度が低い
  投稿: 「バンド上限に到達」という情報は投稿の価値を生みにくい

フィボナッチ
  必要: チャート上の波動認識（人間の判断が必要）
  T-CAX: 数値データだけでは波動を特定できない
  投稿: スプレッドシートに落とし込むのが根本的に困難

一目均衡表
  必要: 26期間以上の複雑な計算 + 雲の視覚的判断
  T-CAX: 計算は可能だが、テキスト投稿で伝えにくい
  投稿: 「雲の上にいるから買い」はコンパナの口調に合わない
```

### 7-3. 結論

```
通貨強弱 + ダウ理論は「データ」「投稿スタイル」「運用」の全てに適合する唯一の組み合わせ。
これ以上インジケーターを足すより、この2つの精度を上げていく方がT-CAXの価値につながる。

  通貨強弱 → 「今何が起きているか」（短期・リアルタイム）
  ダウ理論 → 「この流れは続くのか」（中期・トレンド判定）

この2軸で市場を語れば、テクニカル分析に頼らずとも
「読みやすく、ためになる」投稿が生まれる。
```

## 9. バックフィル機能（日次レートからの遡及計算）

```
関数: backfillCurrencyStrength()（rateManager.gs）
実行: GASエディタから手動で1回だけ実行

処理:
  1. 日次レートの全データ（〜1,341日分）を読み込み
  2. 日ごとに前日終値比で通貨強弱を計算
  3. 勢い・初動・3日トレンド・ダウ理論も計算
  4. 既存の毎時データ（本日分）を退避
  5. 日次データを直近164日分に絞り込み（168行制限 - 毎時データ分）
  6. 日次164行 + 毎時4行 = 168行をシートに書き込み

注意:
  ・23:59の日時は日次バックフィルデータ、それ以外は毎時データとして判別
  ・168行制限は書き込み前に適用（書き込み後の削除は行わない）
  ・日次レートが再構築された場合は再実行で最新データを反映
```

## 10. 運用中に発覚した問題と対応（2026-04-06）

```
【問題1】priceSummary.gsの中身がGAS上で別バージョンになっていた
  影響: aggregateDailyRatesが正しく動かず、日次レートが3/14で止まっていた
  対応: priceSummary.gsを正しいバージョンで上書き

【問題2】日次レート 3/15〜4/4の空白
  原因: 問題1により日次集約が止まっていた + レートキャッシュは7日で自動削除
  対応: forceRebuildDailyRates()でTwelve Data APIから5年分を再取得（1,341日分）
        ※ rebuildDailyRates()はUI関数を使うためGASエディタから実行不可
        → UI不要版のforceRebuildDailyRates()をtestFunctions.gsに追加

【教訓】
  ・GASファイルの中身が意図せず変わる可能性がある → 定期的にバージョン確認
  ・日次レートはforceRebuildDailyRatesで復元可能（Twelve Data APIクレジット7消費）
```
