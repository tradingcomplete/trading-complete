# 水準2-4: プロンプト類似セクション統合 シミュレーション

**作成日**: 2026-04-18  
**目的**: 現在23,869文字・72セクションのプロンプトを、品質を維持しつつ更に削減  
**方針**: 重複と類似のみ統合。必要情報は削らない。LLMのattention最適化を優先。

---

## 1. 前提:LLMのコンテキスト理解

### 1.1 Claude のコンテキスト性能

- Claude Sonnet 4のコンテキスト窓: 200,000トークン
- 現在のプロンプト: 約12,000トークン(24,000字)
- 技術的には十分余裕がある

### 1.2 しかし「長ければ良い」ではない

**研究で確認されている現象**:
- 指示の散乱: 同じルールが複数箇所にあると、LLMは最新の指示を優先し他を無視する
- middle-lost: プロンプト中盤の情報が最も無視されやすい(冒頭と末尾は注目される)
- attention dilution: 総文字数に反比例して各指示の重要度が下がる

### 1.3 TCAXの実測データから

- 朝の WEEKLY_REVIEW(29,686文字): Q6①論理矛盾が修正で見落とされた
- 修正後(23,869文字): Q6.5で検出・修正成功

**長さを減らすほど指示遵守率が上がる**傾向は明確。ただし情報を落とすと品質が下がる。

## 3. 統合案の詳細

### 3.1 統合案A: フォーマット系5セクションを1つに

**Before(5セクション・推定2,500字)**:
```
【フォーマット構造】... 詳細な絵文字ブロック定義
【絵文字の位置】... 行頭ルール
【投稿の構造】... 因果チェーンの骨格
【フォーマットルール】... ハッシュタグ・句点ルール
【絵文字行と→行の書き分けルール】... OK/NG例
```

**After(1セクション・推定1,500字)**:
```
【投稿フォーマット(絶対遵守)】

■ 骨格:原因→経路→為替への影響。冒頭1行が全て。レートの数字で始めるな。

■ 構造:絵文字ブロック 3-4個+締め1-2行
  絵文字行:事実を体言止め・動詞止めで短く(速報ヘッドライン調)。感想は入れるな。
  →行:1ブロックに1つ。背景・分析・感想を1〜2文で完結。
  絵文字:📕📝📋☕💡⚠️✅ の7種のみ。必ず行頭。行末配置は禁止。

■ ルール:
  絵文字ブロック最低3個。0個は禁止。
  話題が変わったら空行+新しい絵文字。
  最後は絵文字なし・→なしで1〜2行の締め。
  ハッシュタグは書くな(自動付与)。ピリオド・URL・マークダウン禁止。句点「。」。
  リスト記号(・●1.-)禁止。

■ OK例:
  ☕パウエル、まだ利下げしない。
  →タカ派的な発言で市場の期待を冷ました感じですね。
  
  📝豪ドルは資源国通貨として原油高の恩恵。
  →豪ドル米ドルは上昇トレンドが加速中ですが、ボラが高まる可能性もありますね。
  
  原油価格と各国の金融政策、答え合わせは今夜ですね。

■ NG:1ブロックに→2個以上/絵文字0個/絵文字行に感想/絵文字を行末に
```

**削減効果**: 約1,000字削減、セクション 5→1

### 3.2 統合案B: TC関連6セクションを1つに

**Before(6セクション・推定2,000字)**:
```
【Trading Complete(TC)について】
【TCが解決するユーザーの課題】
【TCのターゲット層】
【TCの核心的価値】
【AIへの厳格な記述ルール】
【TC開発の背景】
```

**After(1セクション・推定800字)**:
```
【Trading Complete(TC)について】

■ 概要:トレード記録・分析SaaS。「勝つため」ではなく「規律を守るため」のツール。

■ 何を解決するか:
  ・Excelでのトレード記録の限界(継続性・分析の深さ)
  ・エントリー根拠を言語化する習慣がないままの感覚トレード
  ・振り返りが続かず同じ失敗を繰り返す

■ 対象:副業トレーダー(記録を習慣化したい30代の会社員が中心)

■ 核心価値:「記録→分析→改善」のループをツールで仕組み化する

■ 機能例(投稿では1つだけ自然に触れる):
  3根拠記録/ルール遵守バッジ/月別パフォーマンス分析/セッション別統計/Market Notes/確定申告準備

■ 記述トーン:TCは「勝率UPツール」ではない。「規律形成ツール」として自然に触れる。宣伝感厳禁。
```

**削減効果**: 約1,200字削減、セクション 6→1

### 3.3 統合案C: 市場の因果関係3セクション

**Before(3セクション・推定1,200字)**:
```
【WTI原油と為替の因果関係(間違えると致命的)】
【経済指標の方向性ルール(誤解釈は致命的)】
【リスクセンチメントと円の方向性(絶対に間違えるな)】
```

**After(1セクション・推定800字)**:
```
【市場の因果関係(間違えると致命的)】

以下の3点は間違えると投稿が致命的に壊れる。厳守せよ。

■ 原油と通貨:
  原油高 → 資源国通貨(豪ドル・カナダドル)買い・円は輸入コスト増で売られやすい
  原油安 → 資源国通貨売り・円は輸入コスト減で買われやすい
  × NG「原油下落でドル高」「WTI下落で円安」

■ 経済指標:
  通常指標(GDP,PMI,CPI等): 予想より高い=上振れ=買い要因
  逆指標(失業率等): 予想より低い=改善=買い要因
  注入データの(買い/売り要因)判定をそのまま使え

■ リスクセンチメント:
  リスクオフ(戦争・株安)= 安全通貨の円が買われる = 円高
  リスクオン(株高・楽観)= 円が売られる = 円安
  × 絶対NG「リスク回避で円売り」「リスクオフで円安」
```

**削減効果**: 約400字削減、セクション 3→1

### 3.4 統合案D: 事実・論理3セクション

**Before(3セクション・推定700字)**:
```
【環境認識の原則】
【事実とフィクションの区別 / 数値の正確性】
【論理の一貫性】
```

**After(1セクション・推定500字)**:
```
【事実・論理の遵守(最重要)】

■ 環境認識:TCAXは予測システムではなく認識システム。
  データが示す事実がすべて。上昇なら「上昇している」。「下がるかも」と願望を書くな。
  仮説判定(○/△/×)が注入されたら100%従え。判定と逆のことを書くな。

■ 事実性:
  架空のニュース・要人発言・指標日程は絶対禁止。
  為替レートは【本日の確定レート】の数値のみ使え。
  Gemini検索で得た数値は使うな。

■ 指標名:
  CPI(消費者物価)とPPI(生産者物価)は別物。HICPも別。
  カレンダー記載の正式名称をそのまま使え。国名を必ず明記。

■ 因果関係:
  日銀→円 / FRB→ドル / ECB→ユーロ / BOE→ポンド / RBA→豪ドル。間違えるな。
```

**削減効果**: 約200字削減、セクション 3→1

### 3.5 統合案E: 重複削除(純粋に削るだけ)

| 削除対象 | 理由 |
|---|---|
| buildFormatRules 末尾の【ペルソナ】再掲 | 冒頭キャラクター注入で既出 |
| 【コンパナの口調(絶対遵守)】 | 冒頭【発信の原則と口調】と内容重複 |
| 【TCの主な機能】 | 【Trading Completeの主要機能】と同じ(統合案Bに吸収) |
| 末尾の「コンパナの口調で書け」の過剰な繰り返し | 既に核心行だけにしたのでこれ以上削ってもOK |

**削減効果**: 約500字削減、セクション 4個削減

### 4.3 目標未達の評価

診断書目標(15,000字・40セクション)には届かない。

**なぜ届かないか**:
- キャラクターシート内22セクションは削れない(コンパナの特徴そのもの)
- 市場データ11セクションは事実情報
- 条件付き注入4セクションは実際に必要な情報

**正直な見立て**:
- 目標値が野心的すぎた可能性
- 現状の22(キャラクター)+11(データ)+4(条件)=37が**削れない最低ライン**
- つまり全体40セクションは技術的には可能だが、ルール・フォーマット系が3セクション以下になる必要があり、それは情報不足リスク
- **現実的な最適解は50〜55セクション・20,000字前後**
