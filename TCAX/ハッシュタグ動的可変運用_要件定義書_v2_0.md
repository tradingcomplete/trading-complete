# ハッシュタグ動的可変運用 要件定義書 v2.0

**発行日**: 2026-04-28
**対象システム**: TCAX (Trading Complete Auto X)
**対象アカウント**: @Compana_Doppio
**性格**: Phase 9 v1.0 の 4 日間本番観察を経て、判定ロジックを「経済カレンダー基準」から「本文主体」に転換する確定版
**前版**: ハッシュタグ動的可変運用_要件定義書_v1_0.md(2026-04-24)

---

## 0. このドキュメントの位置づけ

### v1.0 → v2.0 の転換理由

Phase 9 v1.0(2026-04-24)を 4 日間本番運用した結果、構造的な不整合が観察された:

#### 観察事例 1: 2026-04-28 LONDON 投稿
- 本文の主題: ルビオ・イラン・豪ドル米ドル戻り売り・明日豪 CPI
- 本文に「日銀」の言及: 0 回
- でもハッシュタグ: **#日銀** が付与
- 原因: 経済カレンダーに日銀会合(本日)が ★重要 として残っており、v1.0 ロジックが拾った

#### 観察事例 2: 2026-04-28 GOLDEN 投稿
- 本文の主題: パウエル最後の会見・FOMC・ウォーシュ
- 本文に「FOMC」「パウエル」: 各 2 回以上
- でもハッシュタグ: **#日銀**(本文と完全ミスマッチ)
- 原因: 本日の経済カレンダー ★重要 が「日銀」優先で、本文主題が無視された

### 問題の構造

v1.0 のロジック:
```
1. getEconomicCalendar_('today') → ★重要 行を抽出
2. 候補タグ配列を生成
3. determinePrimaryEventTag_ で 1 つ決定(本文言及回数も見るが候補は経済カレンダー由来)
```

→ **本文の主題と無関係なタグが付くケースが構造的に発生**

### コンパナさん指針(2026-04-28)

> 「ハッシュタグが普段 0 個、必要に応じて 1 個だが、作成された本文から考えるように」

→ 経済カレンダー連携を撤廃し、**本文主体判定**に転換する。

## 1. v2.0 の判定ロジック

### 動作仕様

```
本文を直接スキャン
  ↓
各イベントタグのキーワード言及回数を集計
  ↓
2 回以上言及されたキーワードのみ候補に
  ↓
候補 0 個 → ハッシュタグなし(0 個)
  ↓
候補 1 個 → そのタグを付与
  ↓
候補 2 個以上 → 優先度テーブル + 言及回数で 1 つ決定
  ↓
末尾に 1 個のみ付与
```

### 判定の中核: なぜ「2 回以上」か

- 1 回の軽い言及(例: 「明日の植田会見」)は本文の主題ではない
- 2 回以上言及されているなら、その投稿の主題として機能している
- 「読者が本文を読んだ後、ハッシュタグで思い出すべき主題」を抽出する基準

### 優先度テーブル(v1.0 と同一)

```
FOMC > 日銀 > 雇用統計 > CPI > PPI > GDP > ECB > BOE > RBA > ISM
```

### 対応イベントタグ(v1.0 と同一・10 種類)

| タグ | 検出キーワード |
|---|---|
| #FOMC | FOMC, パウエル |
| #日銀 | 日銀, BOJ, 植田, 金融政策決定会合 |
| #雇用統計 | 雇用統計, NFP, 非農業部門 |
| #CPI | CPI, 消費者物価 |
| #PPI | PPI, 生産者物価 |
| #GDP | GDP |
| #ECB | ECB, ラガルド |
| #BOE | BOE, ベイリー, 英中銀 |
| #RBA | RBA, 豪準備銀行, 豪中銀 |
| #ISM | ISM |

## 2. 実装

### 変更ファイル

`postFinalizer.gs` のみ

### 関数シグネチャの変更

```javascript
// v1.0
function getImportantEventTag_(today) { ... }

// v2.0
function getImportantEventTag_(bodyText) { ... }
```

### 関数実装(v2.0)

```javascript
function getImportantEventTag_(bodyText) {
  if (!bodyText) return [];

  var eventToTag = [
    { patterns: ['FOMC', 'パウエル'],                     tag: '#FOMC' },
    { patterns: ['日銀', 'BOJ', '植田', '金融政策決定会合'], tag: '#日銀' },
    { patterns: ['雇用統計', 'NFP', '非農業部門'],          tag: '#雇用統計' },
    { patterns: ['CPI', '消費者物価'],                     tag: '#CPI' },
    { patterns: ['PPI', '生産者物価'],                     tag: '#PPI' },
    { patterns: ['GDP'],                                  tag: '#GDP' },
    { patterns: ['ECB', 'ラガルド'],                       tag: '#ECB' },
    { patterns: ['BOE', 'ベイリー', '英中銀'],              tag: '#BOE' },
    { patterns: ['RBA', '豪準備銀行', '豪中銀'],            tag: '#RBA' },
    { patterns: ['ISM'],                                  tag: '#ISM' }
  ];

  var detectedTags = [];
  for (var i = 0; i < eventToTag.length; i++) {
    var entry = eventToTag[i];
    var totalCount = countAnyTag_(bodyText, entry.patterns);
    if (totalCount >= 2) {
      detectedTags.push(entry.tag);
    }
  }
  return detectedTags;
}
```

### 呼び出し側の変更

```javascript
// v1.0
var today = new Date();
var candidateTags = getImportantEventTag_(today);

// v2.0
var candidateTags = getImportantEventTag_(bodyText);
```

### 削除されるロジック

- `getEconomicCalendar_('today')` の呼び出し
- 「★重要」マーク行抽出
- 経済カレンダー由来の候補生成

### 保持される関数(変更なし)

- `determinePrimaryEventTag_`: 候補配列を 1 つに絞るロジック(本文言及+優先度テーブル)
- `countAnyTag_`: キーワード言及回数集計

## 5. リスクと対策

### リスク 1: 本文に明示しない暗黙の主題が反映されない

例: 投稿全体が「植田総裁会見の前置き」だが、本文では「会見」「植田」「日銀」のいずれもない

**対策**: ホットトピック選定で当日の重要イベントは既に考慮されており、結果として本文の主題に反映されるはず。本文に出ないなら、そもそも投稿の主題ではないと考える(設計上の割り切り)。

### リスク 2: キーワード言及が偶然 2 回以上に達してしまう

例: 過去の文脈で「日銀」を 2 回言及したが、メインの主題ではない

**対策**: 観察期間で過剰検出を確認。閾値を 2 → 3 に変える可能性は残す(現状は v2.0 で 2 採用)。

### リスク 3: 優先度テーブルにないイベント(例: パウエル氏「FOMC ではなく直接話法」)が拾えない

**対策**: 検出キーワードを継続的に拡充。本リスクは v1.0 から継続中の課題で、Phase 5 で本文ベースで自動学習する仕組みを別途検討。
