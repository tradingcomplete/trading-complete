**発行日**: 2026-04-24
**対象システム**: TCAX (Trading Complete Auto X)
**対象アカウント**: @Compana_Doppio
**性格**: Q1〜Q4 シミュレーション結果を反映した実装判断確定版

## 0. このドキュメントの位置づけ

### 背景

2026年4月時点、X(旧Twitter)アルゴリズムの最新動向を徹底調査した結果、以下が明らかになった:

- **3個以上のハッシュタグは明確に-40%のリーチペナルティ**(国内実測・業界ベンチマーク一致)
- **2026年1月xAIオープンソース化**により、ハッシュタグは手動特徴量から削除され、Grok Transformer が文脈理解する設計に変更
- **Muskが2024/12に「ハッシュタグ使用停止」を明言**、2025/6から広告では全面禁止
- **日本の FX 系著名アカウント(JFX小林社長・羊飼い・西原・田向等)**は **ほぼハッシュタグなし** で運用

### TCAX の現状と問題

- 現状: `generateDynamicHashtags_` が **毎投稿に最大3個のハッシュタグを自動付与**(#FX固定 + イベント + 通貨ペア)
- 問題: 上記レポートの知見に真っ向から反する運用になっており、リーチを抑制している可能性大

### 採用方針

レポートの「**選択肢D: 動的可変(コンテキスト連動)**」を採用。

- **平常時**: ハッシュタグ 0個
- **重要イベント当日**: 1個のみ付与(#日銀 / #FOMC / #雇用統計 等)

## 1. Q1〜Q4 判断の確定(v1.0)

### Q1: イベント判定の情報源 → **(b) 確定データシートの経済カレンダーから当日高重要度を取得**

#### 採用理由

1. 既存の `getEconomicCalendar_` 関数をそのまま再利用可能
2. コンパナが手動メンテナンスしている **重要度「高」** のラベルが SSOT(唯一の真実)
3. 時間窓判定(本日)が正確・曖昧さゼロ
4. 本文キーワード検出は「過去のイベント言及」も拾ってしまう問題あり

#### 不採用理由

- **(a) 本文キーワード検出**: 「過去の日銀会合言及」でも #日銀 が付与されてしまい、トレンド検索流入の空振りが発生する
- **(c) 投稿タイプ+本文複合**: 判定精度が中途半端で、MORNING タイプで日銀解説があっても拾えないケースが出る

---

### Q2: イベント時間窓の定義 → **(a) 当日のイベント = その日の全投稿に付与(シンプル)**

#### 採用理由

1. 日本の FX トレーダーの検索行動は「イベントの**朝から終日**」動く(発表の2時間限定ではない)
2. 実装シンプル(日付一致のみ・時刻計算不要)
3. 1日1ハッシュタグ候補で、投稿間の一貫性が保たれる
4. 発表前の事前投稿も同じタグで拾われる方が、発信者として強い

#### 不採用理由

- **(b) 前後2時間の精密時間窓**: 日銀決定会合は正式発表時刻が曖昧(12:00〜13:00)で時刻判定が不安定
- **(c) INDICATOR タイプのみ**: 日銀決定会合が MORNING で解説されるケースを取り逃す

#### 留意事項

- 将来 A/B テストで効果が弱いと判明した場合、v2.0 で「前日から付与」や「発表後のみ」等を検討

---

### Q3: 複数候補の優先順位 → **(d) 本文主題優先 + 優先度テーブルをフォールバック**

#### ロジック

```
1. 本文で言及されている候補タグのカウントを取得
2. 最多タグが 2 回以上言及 → その主題タグを採用
3. 最多タグが 1 回以下 or 同数 → 事前定義の優先度テーブルで決定
```

#### 優先度テーブル(v1.0 提案)

```
FOMC > 日銀 > 雇用統計 > CPI > PPI > GDP > ECB > BOE > RBA > ISM > その他
```

**根拠**:
- 米国の金融政策(FOMC)は世界市場への影響度が最大
- 日本語投稿であるため、日銀は FOMC に次ぐ重要度
- 雇用統計・CPI は米国指標として次点
- ECB・BOE・RBA は日本語投稿では相対的に検索ボリューム小

#### 採用理由

- 本文の主題が明確な場合は主題を優先(読者視点で自然)
- 主題が曖昧な場合は世界市場の重要度で判断(検索流入を最大化)

---

### Q4: #FX 固定の扱い → **(a) 完全撤廃・平常時は完全0個**

#### 採用理由

1. レポートの結論「平常時は0個」と完全整合
2. #FX は generic hashtag として「drowned out」カテゴリ(レポート明記)
3. RULE系/週次系を残す案は効果測定の母数不足で判断不能
4. 「平常時0個」で運用して、A/B テスト結果を見て v2.0 で微調整

#### シャドウバン回避効果

- 機械生成ボットは毎投稿同一タグが「スパム判定の教科書的パターン」
- #FX 固定撤廃により、この典型パターンからも脱出

## 2. 実装仕様

### 2-1. 関数構成

```
generateDynamicHashtags_(text, postType, rates)
  ├── [v2.0 新規] getImportantEventTag_(today)
  │     ├── getEconomicCalendar_('today') の結果をパース
  │     ├── 重要度「高」のイベントのみ抽出
  │     └── 該当なし → null を返す
  │
  ├── [v2.0 新規] determinePrimaryEventTag_(bodyText, eventTags)
  │     ├── 本文の言及回数を計測
  │     ├── 最多タグが2回以上 → 主題タグ採用
  │     └── それ以外 → 優先度テーブル順
  │
  └── 旧ロジック(#FX固定、通貨ペア検出、テーマ検出)は全削除
```

### 2-2. `getImportantEventTag_` 仕様

```javascript
/**
 * 本日の重要度「高」イベントから、該当するハッシュタグ候補を抽出する。
 * 
 * @param {Date} today - 判定基準日(通常は new Date())
 * @return {Array<string>} 該当タグ候補配列(例: ['#FOMC', '#日銀'])、該当なしは []
 */
function getImportantEventTag_(today) {
  // getEconomicCalendar_('today') は「今日の経済指標」セクションを含む文字列を返す
  var calendarText = getEconomicCalendar_('today');
  if (!calendarText) return [];
  
  // 「★重要」マークが付いた行のみ抽出
  var importantLines = calendarText.split('\n').filter(function(line) {
    return line.indexOf('★重要') !== -1;
  });
  if (importantLines.length === 0) return [];
  
  // イベント名 → ハッシュタグ のマッピング
  var eventToTag = [
    { patterns: ['FOMC', 'パウエル'],                 tag: '#FOMC' },
    { patterns: ['日銀', 'BOJ', '植田', '金融政策決定会合'], tag: '#日銀' },
    { patterns: ['雇用統計', 'NFP', '非農業部門'],         tag: '#雇用統計' },
    { patterns: ['CPI', '消費者物価'],                    tag: '#CPI' },
    { patterns: ['PPI', '生産者物価'],                    tag: '#PPI' },
    { patterns: ['GDP'],                              tag: '#GDP' },
    { patterns: ['ECB', 'ラガルド'],                      tag: '#ECB' },
    { patterns: ['BOE', 'ベイリー', '英中銀'],             tag: '#BOE' },
    { patterns: ['RBA', '豪準備銀行', '豪中銀'],           tag: '#RBA' },
    { patterns: ['ISM'],                              tag: '#ISM' }
  ];
  
  var detectedTags = [];
  for (var i = 0; i < importantLines.length; i++) {
    var line = importantLines[i];
    for (var j = 0; j < eventToTag.length; j++) {
      var entry = eventToTag[j];
      for (var k = 0; k < entry.patterns.length; k++) {
        if (line.indexOf(entry.patterns[k]) !== -1) {
          if (detectedTags.indexOf(entry.tag) === -1) {
            detectedTags.push(entry.tag);
          }
          break;
        }
      }
    }
  }
  
  return detectedTags;
}
```

### 2-3. `determinePrimaryEventTag_` 仕様

```javascript
/**
 * 複数のイベントタグ候補から、本投稿に付けるべきタグを1つ決定する。
 * 
 * @param {string} bodyText - 投稿本文
 * @param {Array<string>} eventTags - 候補タグ配列
 * @return {string|null} 採用する1個のタグ(該当なしはnull)
 */
function determinePrimaryEventTag_(bodyText, eventTags) {
  if (!eventTags || eventTags.length === 0) return null;
  if (eventTags.length === 1) return eventTags[0];
  
  // タグ → 本文キーワードパターンの逆引き
  var tagToKeywords = {
    '#FOMC':    ['FOMC', 'パウエル'],
    '#日銀':    ['日銀', 'BOJ', '植田', '金融政策決定会合'],
    '#雇用統計': ['雇用統計', 'NFP', '非農業部門'],
    '#CPI':     ['CPI', '消費者物価'],
    '#PPI':     ['PPI', '生産者物価'],
    '#GDP':     ['GDP'],
    '#ECB':     ['ECB', 'ラガルド'],
    '#BOE':     ['BOE', 'ベイリー', '英中銀'],
    '#RBA':     ['RBA', '豪準備銀行', '豪中銀'],
    '#ISM':     ['ISM']
  };
  
  // Step 1: 本文での言及回数をカウント
  var tagCounts = eventTags.map(function(tag) {
    var keywords = tagToKeywords[tag] || [];
    var count = 0;
    for (var i = 0; i < keywords.length; i++) {
      var re = new RegExp(keywords[i], 'g');
      var m = bodyText.match(re);
      count += m ? m.length : 0;
    }
    return { tag: tag, count: count };
  });
  
  // Step 2: 最多タグが 2 回以上言及されていれば主題として採用
  tagCounts.sort(function(a, b) { return b.count - a.count; });
  if (tagCounts[0].count >= 2 && tagCounts[0].count > tagCounts[1].count) {
    return tagCounts[0].tag;
  }
  
  // Step 3: 主題が曖昧 → 優先度テーブルで決定
  var priorityOrder = ['#FOMC', '#日銀', '#雇用統計', '#CPI', '#PPI', '#GDP', '#ECB', '#BOE', '#RBA', '#ISM'];
  for (var j = 0; j < priorityOrder.length; j++) {
    if (eventTags.indexOf(priorityOrder[j]) !== -1) {
      return priorityOrder[j];
    }
  }
  
  return eventTags[0]; // フォールバック
}
```

### 2-4. `generateDynamicHashtags_` 新仕様

```javascript
/**
 * ★v15.0 Phase 9 v1.0 (2026-XX-XX): ハッシュタグ動的可変運用
 * 
 * 方針:
 *   - 平常時: ハッシュタグ0個
 *   - 重要イベント当日: 1個のみ (#FOMC / #日銀 等)
 * 
 * 削除された旧ロジック:
 *   - #FX 固定付与
 *   - 通貨ペア検出(#ドル円 等)
 *   - テーマ検出(#損切り 等)
 *   - フォールバック(#為替 / #トレード)
 *   - 3個上限ロジック
 */
function generateDynamicHashtags_(text, postType) {
  var lines = text.split('\n');
  
  // === Step 1: Claude/Geminiが付けた既存ハッシュタグ行を除去 ===
  var cleanLines = [];
  for (var i = 0; i < lines.length; i++) {
    var trimmed = lines[i].trim();
    // ハッシュタグのみの行を除去
    if (trimmed.length > 0 && trimmed.length < 100 && /^#\S+(\s+#\S+)*$/.test(trimmed)) {
      continue;
    }
    cleanLines.push(lines[i]);
  }
  
  // 末尾の空行を整理
  while (cleanLines.length > 0 && cleanLines[cleanLines.length - 1].trim() === '') {
    cleanLines.pop();
  }
  
  var bodyText = cleanLines.join('\n');
  
  // === Step 2: 本日の重要イベントを確認 ===
  var today = new Date();
  var candidateTags = getImportantEventTag_(today);
  
  if (candidateTags.length === 0) {
    // 平常時 → ハッシュタグなし
    console.log('🏷️ ハッシュタグ: なし (平常時)');
    return bodyText;
  }
  
  // === Step 3: 複数候補から1つを決定 ===
  var primaryTag = determinePrimaryEventTag_(bodyText, candidateTags);
  
  if (!primaryTag) {
    console.log('🏷️ ハッシュタグ: なし (候補はあるが判定失敗)');
    return bodyText;
  }
  
  // === Step 4: 1個のみ付与 ===
  console.log('🏷️ ハッシュタグ自動生成: ' + primaryTag + ' (重要イベント当日)');
  return bodyText + '\n\n' + primaryTag;
}
```

## 6. 動作確認シナリオ

### シナリオ1: 平常時(経済指標なし)

- **入力**: 通常の MORNING 投稿本文(ドル円の値動き分析)
- **期待**: ハッシュタグなし(本文のみ)
- **確認ログ**: `🏷️ ハッシュタグ: なし (平常時)`

### シナリオ2: FOMC 当日(単独イベント)

- **入力**: FOMC に言及した MORNING 投稿
- **経済カレンダー**: 本日 22:30 FOMC ★重要
- **期待**: `#FOMC` が末尾に1個付与

### シナリオ3: 日銀+米雇用統計同日

- **入力**: 本文で日銀を3回、雇用統計を1回言及
- **経済カレンダー**: 本日 12:00 日銀 ★重要、21:30 米雇用統計 ★重要
- **期待**: 主題(言及回数)で日銀が優先 → `#日銀` が付与
- **確認ログ**: `🏷️ ハッシュタグ自動生成: #日銀 (重要イベント当日)`

### シナリオ4: FOMC+日銀同日、本文で両方同数言及

- **入力**: FOMC を2回、日銀を2回言及
- **経済カレンダー**: 両方 ★重要
- **期待**: 優先度テーブルで FOMC > 日銀 → `#FOMC` が付与

### シナリオ5: 指標あるが重要度「中」のみ

- **経済カレンダー**: 本日 新規失業保険申請件数(重要度 中)のみ
- **期待**: ハッシュタグなし(★重要マークがないため)

### シナリオ6: 3月17日(RBA利上げ当日)

- **本文**: RBA 連続利上げ について MORNING で言及
- **経済カレンダー**: RBA 政策金利発表 ★重要
- **期待**: `#RBA` が付与

### 7-2. 測定指標

| 指標 | 測定方法 | 期待値(レポート推定) |

## 8. 将来の v2.0 / v3.0 検討項目(今回のスコープ外)

- **日銀会合の前日からタグ付与**: 前日の MORNING から「明日は日銀」として #日銀 を付与
- **発表後のみ付与(事後モード)**: 発表時刻以降の投稿のみに付与
- **トランプ関税・為替介入等の突発イベント**: 経済カレンダーに登録されていない突発イベントへの対応
- **キャンペーン独自タグ**: #TCAX朝ドル 等のブランドタグ運用(企画次第)
- **画像添付・リプライ誘発等のアルゴリズム対策**: ハッシュタグ以外の強化要素(レポート §6 参照)

## 9. リスクと対策

### 9-1. リスク

1. **経済カレンダーシートの更新漏れ**: コンパナが「★重要」マークを付け忘れると、イベント日でも0個になる
   - 対策: 定期的な経済カレンダーのメンテナンス運用・週次チェックリスト追加
2. **新興イベントへの非対応**: 経済カレンダーに登録されていない急変(為替介入・関税発表等)は対応できない
   - 対策: v2.0 で突発イベント検出ロジックを検討
3. **A/Bテストの交絡要因**: 市場変動・他のプロンプト改善との同時進行で効果を切り分けられない可能性
   - 対策: 他の大きな変更と同時実施を避ける・4週間を必ず確保

### 9-2. ロールバック計画

3段階:
1. **最速**: `getImportantEventTag_` が常に [] を返すようにする(0個運用にロック)
2. **中速**: `generateDynamicHashtags_` を旧版(3個固定)に戻す
3. **完全**: バックアップファイルから復元
